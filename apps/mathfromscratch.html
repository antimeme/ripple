Math From Scratch

If you want to make an apple pie from scratch, you must first create the universe. — Carl Sagan

Mathematics has no generally accepted definition. We could describe it as a collection of tools for understanding patterns and for studying structure and change. People have studied mathematics for centuries and found applications for it in every field of science and technology.

This is an attempt to build the most important parts of math from the simplest possible foundations. Forget everything you know about math and let's begin at the beginning.

We need a foundation on which to build so we'll use one created by Zermelo and Fraenkel from 1908 to 1921. Let's suppose that an object called a set exists. Sets can contain other sets and they obey the following axioms:

  • Extensionality: two sets are equal if they contain the same elements.
    ∀a,b : ∀c (c ∈ a ⟺ c ∈ b) ⟹ a = b
    • Reflexive: ∀a : a = a
    • Symmetric: ∀a,b : (a = b) ⟺ (b = a)
    • Transitive: ∀a,b,c : (a=c ∧ b=c) ⟹ (a = b)
    This means sets have no order because, for example, {a, b} and {b, a} have the same elements and therefore are the same set.
  • Pairing: for any two sets there is a third set which contains only those two as members
    ∀a,b ∃{a, b} ∀c (c∈{a, b} ⟹ (c = a ∨ c = b))
  • Union: for any set there eixsts a set which contains all members found within the members of the set.
    ∀a ∃b ∀c (c∈b ⟹ ∃d∈a : c∈d)
    For example, if the set {{a, b}, {b, c}} exists then the union of its members, ∪{{a, b}, {b, c}} which is {a, b, c}, also exists.
  • Power Set: for any set there exists another set which contains all subsets of that set.
    ∀a ∃b ∀c c∈b ⟹ c⊆a
    So if the set {a, b, c} exists then a power set containing all subsets of it, which is {{}, {a}, {b}, {c}, {a, b}, {b, c}, {a, c}, {a, b, c}}, also exists.
  • Regularity: if a set contains any elements then it contains at least one element that shares no members with itself.
    ∀a ∃b∈a ⟹ ∃c∈a ∄d (d∈a ∧ d∈c)
    No set can contain itself because of this.
  • Specification: given a set and a predicate the subset containing only elements for which the predicate is true exists.
    ∀a ∃b⊆a ∀c (c∈b ⟺ ∃d : d∈a ∧ f(d))
  • Replacement
  • Infinity
  • Choice

Ordered Pairs

Extensionality means that two sets which contain the same elements are the same set, so members of a set have no meaningful order.

∀a,b : {a, b} = {b, a}

Let's impose some order on them.

  • Kuratowski 1921: (a, b) ≝ {{a}, {a, b}}
  • a is the first element of (a, b) ≝ ∀c∈(a, b) : a∈c
  • b is the second element of (a, b) ≝
    (∃c∈(a, b) : b∈c) ∧ (∀c,d∈(a, b) : c ≠ d ⟹ b≠c ∧ b≠d)
  • Not reversible (unlike sets): ∀a,b (a ≠ b) ⟹ ((a, b) ≠ (b, a))
    {{a}, {a, b}} ≠ {{b}, {a, b}}
  • Extend by nesting: ((a, b), c)

Cardinality

  • Relation: (r : D ⟶ R) ≝ { (a, b) : a∈D,b∈R }
    • D is the Domain of the Relation
    • R is the Range of the Relation
  • Application: r(a) = b ⟺ (a, b)∈r
  • Given relation r : D ⟶ R
    • Injection: ∀a,b∈D : r(a) = r(b) ⟹ a = b
      No two elements of D map the the same element of R
    • Surjection: ∀a∈R ∃b∈D : r(b) = a
      Every element of R can be reached by at least one element of D
    • Bijection: injection and surjection
  • Two sets with a bijection have the same cardinality
    #{a, b, c} = #{d, e, f}
    r = {(a, d), (b, e), (d, f)}

Addition ℕ

So far there's nothing we can do with natural numbers except to compare cardinality with other sets. Let's define a binary operation which maps a pair of natural numbers to another natural number. We will call this operation addition and represent it with a + sign between two natural numbers like this: a + b

  • Definition (Grassman 1861):
    • Base: ∀a∈ℕ a + 0 ≝ a
    • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)
  • Example: 3 + 2
    • 3 + 2 = 3 + succ(1) :: Definition of 2
    • 3 + 2 = 3 + succ(succ(0)) :: Definition of 1
    • 3 + 2 = succ(3 + succ(0)) :: Definition Step
    • 3 + 2 = succ(succ(3 + 0)) :: Definition Step
    • 3 + 2 = succ(succ(3)) :: Definition Base
    • 3 + 2 = succ(4) :: Successor
    • 3 + 2 = 5 :: Successor
  • Closed: ∀a,b∈ℕ : a + b ∈ℕ
  • Associative: ∀a,b,c∈ℕ : (a + b) + c = a + (b + c)
    • (a + b) + 0 = a + b = a + (b + 0) :: Definition Base
    • ∃c∈ℕ ∀a,b∈ℕ : (a + b) + c = a + (b + c) :: Assumption
    • (a + b) + succ(c) = succ((a + b) + c) :: Definition Step
    • (a + b) + succ(c) = succ(a + (b + c)) :: Induction
    • (a + b) + succ(c) = a + succ(b + c) :: Definition Step
    • (a + b) + succ(c) = a + (b + succ(c)) :: Definition Step
  • Identity: ∀a∈ℕ a + 0 = a = 0 + a
    • 0 + 0 = 0 :: Definition Base
    • ∃a∈ℕ : 0 + a = a :: Assumption
    • 0 + succ(a) = succ(0 + a) :: Definition Step
    • 0 + succ(a) = succ(a) :: Induction
  • Commutative: ∀a,b∈ℕ : a + b = b + a
    • Successor + 1: ∀a∈ℕ succ(a) = a + 1
      • succ(a) = succ(a + 0) (Definition Base)
      • succ(a) = a + succ(0) (Definition Step)
      • succ(a) = a + 1 (Definition of 1)
    • Commutativity of 1: ∀a∈ℕ a + 1 = 1 + a
      • 0 + 1 = 1 + 0 :: Identity
      • ∃a∈ℕ : a + 1 = 1 + a :: Assumption
      • succ(a) + 1 = succ(a) + succ(0) :: Definition of 1
      • succ(a) + 1 = succ(succ(a) + 0) :: Defintion Step
      • succ(a) + 1 = succ((a + 1) + 0) :: Successor + 1
      • succ(a) + 1 = succ(a + 1) :: Defintion Base
      • succ(a) + 1 = succ(1 + a) :: Induction
      • succ(a) + 1 = 1 + succ(a) :: Definition Step
    • ∃b∈ℕ ∀a∈ℕ : a + b = b + a :: Assumption
    • a + succ(b) = a + (b + 1) :: Successor + 1
    • a + succ(b) = (a + b) + 1 :: Associativity
    • a + succ(b) = (b + a) + 1 :: Induction
    • a + succ(b) = b + (a + 1) :: Associativity
    • a + succ(b) = b + (1 + a) :: Commutativity of 1
    • a + succ(b) = (b + 1) + a :: Associativity
    • a + succ(b) = succ(b) + a :: Successor + 1

Multiplication ℕ

Adding natural numbers is useful but we will define another binary operation called multiplication which is also a binary operation. It maps a pair of natural numbers to another natural number, just like addition but is defined differently. We will represent it with a × sign between two natural numbers like this: a × b

  • Definition (Grassman 1861):
    • Base: ∀a∈ℕ a × 0 ≝ 0
    • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)
  • Example: 3 × 2
    • 3 × 2 = 3 × succ(1) :: Definition of 2
    • 3 × 2 = 3 × succ(succ(0)) :: Definition of 1
    • 3 × 2 = 3 + (3 × succ(0)) :: Definition Step
    • 3 × 2 = 3 + (3 + (3 × 0)) :: Definition Step
    • 3 × 2 = 3 + (3 + 0) :: Definition Base
    • 3 × 2 = 3 + 3 :: Addition
    • 3 × 2 = 6 :: Addition
  • Closed: ∀a,b∈ℕ : a × b ∈ℕ
  • Right Distributive: ∀a,b,c∈ℕ :
    (a + b) × c = (a × c) + (b × c)
    • (a + b) × 0 = (a × 0) + (b × 0)
      • (a + b) × 0 = 0 :: Definition Base
      • (a + b) × 0 = 0 + 0 :: Definition Addition
      • (a + b) × 0 = (a × 0) + (b × 0) :: Definition Base
    • ∃c∈ℕ ∀a,b∈ℕ : ((a + b) × c = (a × c) + (b × c)) :: Assumption
    • ((a + b) × succ(c) = (a + b) + ((a + b) × c) :: Definition Step
    • ((a + b) × succ(c) = (a + b) + ((a × c) + (b × c)) :: Induction
    • ((a + b) × succ(c) = (a + (a × c)) + (b + (b × c)) :: Associative and Commutative Addition
    • ((a + b) × succ(c) = (a × succ(c)) + (b × succ(c)) :: Definition Step
  • Left Distributive: ∀a,b,c∈ℕ :
    c × (a + b) = (c × a) + (c × b)
    • c × (a + 0) = (c × a) + (c × 0)
      • c × (a + 0) = c × a :: Definition Addition
      • c × (a + 0) = (c × a) + 0 :: Definition Addition
      • c × (a + 0) = (c × a) + (c × 0) :: Definition Base
    • ∃c∈ℕ ∀a,b∈ℕ : c × (a + b) = (c × a) + (c × b) :: Assumption
    • c × (a + succ(b)) = c × succ(a + b) :: Definition Addition
    • c × (a + succ(b)) = c + (c × (a + b)) :: Definition Step
    • c × (a + succ(b)) = c + ((c × a) + (c × b)) :: Induction
    • c × (a + succ(b)) = (c × a) + (c + (c × b)) :: Associative and Commutative Addition
    • c × (a + succ(b)) = (c × a) + (c × succ(b)) :: Definition Step
  • Distributive:
    ((a + b) × c = (a × c) + (b × c)) ∧
    (c × (a + b) = (c × a) + (c × b))
  • Associative: ∀a,b,c∈ℕ : (a × b) × c = a × (b × c)
    • (a × b) × 0 = a × (b × 0)
      • (a × b) × 0 = 0 :: Definition Base
      • (a × b) × 0 = a × 0 :: Definition Base
      • (a × b) × 0 = a × (b × 0) :: Definition Base
    • ∃c∈ℕ ∀a,b∈ℕ : (a × b) × c = a × (b × c) :: Assumption
    • (a × b) × succ(c) = (a × b) + ((a × b) × c) :: Definition Step
    • (a × b) × succ(c) = (a × b) + (a × (b × c)) :: Induction
    • (a × b) × succ(c) = a × (b + (b × c)) :: Left Distributive
    • (a × b) × succ(c) = a × (b × succ(c)) :: Definition Step
  • Identity: ∀a∈ℕ a × 1 = a = 1 × a
    • a × 1 = a × succ(0) :: Definition of 1
    • a × 1 = a + (a × 0) :: Definition Step
    • a × 1 = a + 0 :: Definition Base
    • a × 1 = a :: Definition Addition
    • 1 × 0 = 0 :: Definition Base
    • ∃a∈ℕ 1 × a = a :: Assumption
    • 1 × succ(a) = 1 + (1 × a) :: Definition Step
    • 1 × succ(a) = 1 + a :: Induction
    • 1 × succ(a) = succ(a) :: Addition
  • Commutative: ∀a,b∈ℕ : a × b = b × a
    • a × 0 = 0 × a
      • 0 × 0 = 0 :: Definition Base
      • ∃a∈ℕ 0 × a = 0 :: Assumption
      • 0 × succ(a) = 0 + (0 × a) :: Definition Step
      • 0 × succ(a) = 0 + 0 :: Induction
      • 0 × succ(a) = 0 :: Addition
    • ∃b∈ℕ ∀a∈ℕ : a × b = b × a :: Assumption
    • a × succ(b) = a + (a × b) :: Definition Step
    • a × succ(b) = a + (b × a) :: Induction
    • a × succ(b) = (b × a) + a :: Commutative Addition
    • a × succ(b) = (b × a) + (1 × a) :: Identity
    • a × succ(b) = (b + 1) × a :: Left Distributive
    • a × succ(b) = succ(b) × a :: Successor + 1

Integers ℤ

Natural numbers are useful but they don't have additive or multiplicative inverses. There's no natural number we can add to 2 in order to get zero, which means we can't undo an operation. This makes natural numbers unsuitable for representing concepts like debt. We can create extended numbers that support values less than zero. We'll call these numbers integers and use the ℤ symbol to represent the set of all integers.

An integers is an ordered pair of natural numbers. Unlike natural numbers which are unique sets, there are an infinite number of ordered pairs of natural numbers which can represent the same integer. This means we need a new notion of equality which is different from set equality.

  • ℤ ≝ { ∀a,b∈ℕ : (a, b) }

    ((a1,a2) =ℤ (b1,b2)) ≝ a1 + b2 = a2 + b1

    • Symmetric: ∀ (a1,a2)∈ℤ, (b1,b2)∈ℤ :
      a1 + b2 = b1 + a2 ⟺ b1 + a2 = a1 + b2
    • Transitive: TODO
  • Order: ∀ (a1,a2)∈ℤ, (b1,b2)∈ℤ
    ((a1,a2) < (b1,b2)) ≝ a1 + b2 < a2 + b1
  • Equivalence: [a - b] ≝ {∀c∈ℤ : c =ℤ (a, b)}
  • Example: 3ℤ = (3, 0) = {{3}, {3, 0}} = {{{{}, {{}}, {{}, {{}}}}}, {{{}, {{}}, {{}, {{}}}}, {}}}

Addition ℤ

We defined addition for natural numbers but we would like to be able to add integers as well. We can define integer addition in terms of addition of the natural numbers that compose integers. Unlike natural number addition, integer addition has an inverses.

  • Definition: ∀ (a1,a2)∈ℤ, (b1,b2)∈ℤ (a1,a2) + (b1,b2) ≝ (a1 + b1, a2 + b2)
  • Example: 3ℤ + 2ℤ = (3, 0) + (5, 3) = (3 + 5, 0 + 3) = (8, 3) = 5ℤ
  • Closed: ∀a,b∈ℤ : a + b ∈ℤ
  • Associative: ∀a,b,c∈ℤ : (a + b) + c = a + (b + c)
    • ((a1, a2) + (b1, b2)) + (c1, c2) = ((a1 + b1) + c1, (a2 + b2) + c2) :: Definition
    • ((a1, a2) + (b1, b2)) + (c1, c2) = (a1 + (b1 + c1), a2 + (b2 + c2)) :: Associative Addition ℕ
    • ((a1, a2) + (b1, b2)) + (c1, c2) = (a1, a2) + ((b1, b2) + (c1, c2)) :: Definition
  • Identity: ∀a∈ℤ a + 0 = a = 0 + a
    • (a1, a2) + (0, 0) = (a1 + 0, a2 + 0) :: Definition
    • (a1, a2) + (0, 0) = (a1, a2) :: Addition Identity ℕ
    • (a1, a2) + (0, 0) = (0 + a1, 0 + a2) :: Addition Identity ℕ
    • (a1, a2) + (0, 0) = (0, 0) + (a1, a2) :: Definition
  • Commutative: ∀a,b∈ℤ : a + b = b + a
    • (a1, a2) + (b1, b2) = (a1 + b1, a2 + b2) :: Definition
    • (a1, a2) + (b1, b2) = (b1 + a1, b2 + a2) :: Commutative Addition ℕ
    • (a1, a2) + (b1, b2) = (b1, b2) + (a1, a2) :: Definition
  • Inverses: ∀(a1, a2)∈ℤ -(a1, a2) ≝ (a2, a1) : a + (-a) = 0

Rational Numbers ℚ

Integers are a useful extension of natural numbers, but while they introduce additive inverses they don't have multiplicative inverses. Rational numbers have inverses for both addition and multiplication with one important exception. The additive identity has no multiplicative inverse. This happens because all rational numbers multiplied by zero collapse into zero. There's no way to reverese this since the information about which number was multiplied by zero has been lost.

A rational number is an ordered pair of natural numbers. Just as there are an infinite number of ordered pairs of natural numbers which represent the same integer, there are an infinite number of ordered pairs of integers which represent the same natural number. Once again we need a new notion of equality, which is different from integer equality.

    • Symmetric: ∀ (a1,a2)∈ℤ, (b1,b2)∈ℤ :
      a1 × b2 = b1 × a2 ⟺ b1 × a2 = a1 × b2
    • Transitive: TODO

We will extend addition to the rational numbers by adding the integers that make up the rational numbers being added. All of the properties of integers addition will be preserved, including inverses.

  • Definition: ∀a,b∈ℚ a + b ≝ (a1, a2) + (b1, b2) =

  • Associative: ∀a,b,c∈ℚ : (a + b) + c = a + (b + c)
  • Commutative: ∀a,b∈ℚ : a + b = b + a
  • Identity: ∀a∈ℚ a + 0 = a
  • Inverses: ∀a∈ℚ : -a ≝ (-a1, a2) : a = (a1, a2) ∧ a + (-a) = 0

Multiplication ℚ

We will extend multiplication to the rational numbers by multiplying the integers that make up the rational numbers in question. All of the properties of integers multiplication will be preserved. Unlike integers, rational number multiplication has invereses. The exception is zero, which is the additive inverse. Because multiplication by zero takes all numbers to zero there's no way to go backwards from zero.

  • Definition: ∀a,b∈ℚ a × b ≝ (a1, a2) × (b1, b2) = (a1 × b1, a2 × b2)
  • Closed: ∀a,b∈ℚ : a × b ∈ℚ
  • Associative: ∀a,b,c∈ℚ : (a × b) × c = a × (b × c)
  • Commutative: ∀a,b∈ℚ : a × b = b × a
  • Distributive: ∀a,b,c∈ℚ :
    (a × (b + c) = (a × b) + (a × c)) ∧
    ((b + c) × a = (b × a) + (c × a))
  • Identity: ∀a∈ℚ a × 1 = a
  • Inverses: ∀a∉[0 / 1]∈ℚ : a-1 ≝ (a2, a1) : a = (a1, a2) ∧ a × a-1 = 1

Real Numbers ℝ

Rational numbers can represent most quantities that are important but there are some important exceptions. These exceptions are called irrational numbers because they are not rational.

Irrational numbers include almost all roots. Roots are numbers which can be multiplied by themselves some number of times to produce rational numbers. Some roots are rational. For example, 2 × 2 = 4, which means that 2 is the square root of 4. Similarly, 3 × 3 × 3 = 27, which means 3 is the cube root of 27. However, there is no rational number that can be multiplied by itself to get 2.

Suppose this was not true. If there is a rational number which when multiplied by itself results in the rational numbers 2, which can be represented by a pair of integers such as (2, 1) or (416, 208). This means there is a pair of integers, (a, b), such that (a × a, b × b) is equal to (2, 1).

All integers are either even or odd. If an integer a is even then there is another integer a' which can be multiplied by 2 to produce a. For example, 6 is even and 3 can be multiplied by 2 to produce 6. Given a rational number (a, b) which when multiplied by itself equals rational number 2 then we can ask whether a and b are even or odd. If both a and b are even, then we can replace them with a' and b' such that a' × a' = a and b' × b' = b. (a', b') = (a, b) so (a', b') × (a', b') = (2, 1). We can repeat this until either a or b is odd.

According to the definition of rational number multiplication, integers a × a = 2 × b × b; This means a × a must be even, since b × b multiplied by 2 is equal to it. Any odd number multiplied by an odd number is odd, so a must also be even. That means there is an integer a' which when multiplied by 2 is a. So we can replace a with 2 × a': (2 × a') × (2 × a') = 2 × b × b; Multiplication is commutative so: 2 × 2 × a' × a' = 2 × b × b; We can cancel 2 from both sides so: 2 × a' × a' = b × b. This means b × b is even, which means b is even. So a and b are both even, despite what we did earlier to ensure that at least one is odd. This is a contradiction so there can't be a rational number that satisfies this equation.

No rational number can be the root of 2 but we can get arbitrarily close. Let's start with 1. 1 × 1 = 1, which is smaller than 2. We can multiply 2 by 2, but this is 4 which is larger than 2. We can try (3, 2) which squares to (9, 4) but that's larger than (8, 4) which is equal to 2. So we could try (5, 4) which squares to (25, 16) but that's smaller than (32, 16), which is equal to 2. We can continue like this indefinitely. We'll never reach 2 exactly but we'll get closer and closer. In fact we can separate the rational numbers into all the numbers larger than the square root of 2 and all the numbers smaller. This is called a Dedekind cut.

  • Definition: A real number is a cut (Dedekind 1858-1862) which separates all rational numbers into one of two sets. One set contains all the rational numbers equal to or larger than the number we're looking for. The other contains all rational numbers smaller than it. Every rational number can be represented by a real number, but every irrational number can also be represented this way.
  • There are more real numbers than natural numbers, integers or rational numbers (TODO describe Cantor diagonalization)
  • Unlike other kinds of numbers we've described we will not be able to explicitly manipulate real numbers. We can manipulate the rational parts using addition and multiplication but the irrational parts must be carried around. For example, we could talk about 1 + √2. We can't simplify this any further. However, if we multiply it by 1 - √2 we get 1 + √2 × √2 which is 1 + 2 = 3.

Convergent Series

Many real numbers can be represented as a sum of an infinite number of rational numbers. No one has time to add these all up but we can compute a rational number arbitrarily close to the real number by adding enough terms from the series.

Chapter 2: Abstract Algebra

Magma (M, ∘)

A magma is a set M and a binary operation M ∘ M ⟹ M with one important property:

  • Closed: ∀a,b∈M : a ∘ b ∈M

Groups (G, ∘)

A group is a set G and a binary operation G ∘ G ⟹ G with four properties:

  • Closed: ∀a,b∈G : a ∘ b ∈G
  • Associative: (a ∘ b) ∘ c = a ∘ (b ∘ c)
  • Identity: ∃e∈G : ∀a∈G a ∘ e = a
  • Inverses: ∀a∈G ∃a-1∈G : a ∘ a-1 = e

The definition of a group requires that the identity and inverses work when applied from the right. Without knowing anything else about the group we can prove that these values work from the left as well.

  • e = a-1 ∘ (a-1)-1 (Inverses)
  • e = (a-1 ∘ e) ∘ (a-1)-1 (Identity)
  • e = (a-1 ∘ (a ∘ a-1)) ∘ (a-1)-1 (Inverses)
  • e = ((a-1 ∘ a) ∘ a-1) ∘ (a-1)-1 (Associativity)
  • e = (a-1 ∘ a) ∘ (a-1 ∘ (a-1)-1) (Associativity)
  • e = (a-1 ∘ a) ∘ e (Identity)

So inverses work regardless of which side they're applied. This means the identity also works from both sides:

  • e ∘ a = (a ∘ a-1) ∘ a (Inverses)
  • e ∘ a = a ∘ (a-1 ∘ a) (Associativity)
  • e ∘ a = a ∘ e (Inverses)
  • e ∘ a = a (Identity)

A group is the simplest mathematical structure that allows algebra. Suppose we have know a, b and x ∘ a = b:

  • (x ∘ a) ∘ a-1 = b ∘ a-1 (Transitive Equality)
  • x ∘ (a ∘ a-1) = b ∘ a-1 (Associativity)
  • x ∘ e = b ∘ a-1 (Inverses)
  • x = b ∘ a-1 (Identity)

Abelian Groups

An abelian group is a group with a commutative operation. Integers with addition form an abelian group: (ℤ, +).

Rings (R, ∘, ⊚)

A ring is a set and two binary operations R ∘ R ⟹ R and R ⊚ R ⟹ R with four properties:

  • (R, ∘) is an abelian group
  • Closed: ∀a,b∈R : a ⊚ b ∈ R
  • Associative: (a ⊚ b) ⊚c = a ⊚ (b ⊚ c)
  • Distributive: ∀a,b,c∈R :
    (a ⊚ (b ∘ c) = (a ⊚ b) ∘ (a ⊚ c)) ∧
    ((b ∘ c) ⊚ a = (b ⊚ a) ∘ (c ⊚ a))

Commutative Rings

A commutative ring is a with a commutative operations. Integers with addition and multiplication form a commutative ring (ℤ, +, ×).

Unitary Rings (U, ∘, ⊚)

A unitary ring is a ring with an identity element for the second operation:

  • (U, ∘, ⊚) is a ring
  • Identity: ∃e∈U ∀a∈U : a ⊚ e = a

Integers with addition and multiplication form a unitary ring (ℤ, +, ×).

Fields (F, ∘, ⊚)

A field is a unitary ring with inverses:

  • (F, ∘, ⊚) is a unitary ring
  • Inverses: ∀a∈U a ≠ e∘ ⟹ ∃a-1∈U : a ⊚ a-1 = e⊚

An example of a field is rational numbers with addition and multiplication: (ℚ, +, ×)
