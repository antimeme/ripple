Math From Scratch

If you want to make an apple pie from scratch, you must first create the universe. — Carl Sagan
Creating the universe is rather a tall order. We'll have to settle for creating mathematics.

Mathematics has no generally accepted definition. We could describe it as a collection of tools for understanding patterns and for studying structure and change. People have studied mathematics for centuries and found applications for it in every field of science and technology.

We'll do that by stealing the best ideas presented by people around the world over thousands of years.

This is an attempt to build familiar mathematics from the simplest possible foundations. Forget everything you know and let's begin at the beginning.

For the purpose of this presentation, forget everything you know about math.

Logic

Contrariwise, continued Tweedledee, if it was so, it might be; and if it were so, it would be; but as it isn't, it ain't. That's logic. — Through the Looking Glass by Lewis Carroll
Just in case Tweedledee's explanation isn't clear to everyone, let's go over a few things. Logic is complex and addresses many concepts in the wider world. We only need a few relatively simple ideas here.

Predicate

A predicate is a statement that may be true or false based on the value of its variables.

  • Example: _____ is a fruit
  • An apple is a fruit (true)
  • A carrot is a fruit (false)
A predicate is a statement with one or more holes in it. Depending on what we put in the holes it may be true or false.

Compound Predicates

Given predicates P and Q the following are also predicates:

¬P not P (P is false)
(P ∧ Q) both P and Q are true
(P ∨ Q) at least one of P and Q are true
(P ⟹ Q) if P is true then Q is true
(P ⟺ Q) if and only if P is true then Q is true
Once we have predicates we can combine them to form compound predicates. We can invert one with the not operator. We can assert that two prediates are both true using and. We can assert that at least one of two prediates is true using or. We can assert that one predicate follows from another with the double line arrow symbol. We can assert that two predicates are equivalent, meaning they are true or false together, using the two sided double arrow.

Bound Variable Predicates

If P is a predicate with variable a then the following are also predicates:

∀a P P is true for all values of a
∃a P P is true for some value of a
∄a P P is false for all values of a
These are tools we can use to bind variables. The upside down A asserts that the predicate is true for all possible values of the variable. The backwards E asserts that there is at least one value for which the predicate is true. A backwards E with a line through it is the reverse of that: there is no value that makes the predicate true.

Deduction

Deduction applies general rules to reach specific conclusions.

  • Modus ponens: ((P ⟹ Q) ∧ P) ⟹ Q
  • Modus tollens: ((P ⟹ Q) ∧ ¬Q) ⟹ ¬P
  • Syllogism: ((P ⟹ Q) ∧ (Q ⟹ R)) ⟹ (P ⟹ R)
Deduction allows us to derive specific conlusions from general rules. These are some deductive rules.

Induction

Induction uses specific statements to prove more general conclusions. A base establishes a starting point and a step can be repeated indefinitely.

  • P(a) ∧ (P(b) ⟹ P(next(b)))
  • P(b) is true for b after a
Induction allows us to build general rules from specific starting points. We show that if a predicate is true for one value then it must be true for some next value. Combine this with a starting point and we can implicitly repeat the steps as many times as necessary.

Equality

=is equal to (a = b) ⟹ P(a) = P(b)
≠is not equal to (a ≠ b) ≝ ¬(a = b)
  • Reflexive: ∀a (a = a)
  • Symmetric: ∀a,b (a = b) ⟺ (b = a)
  • Transitive: ∀a,b,c (a = b ∧ b = c) ⟹ (a = c)
Equality allows us to substitude one value for another without changing the truth of any predicates. For this to work we need three properties. Reflexive means everything is always equal to itself. Symmetric means equality always goes both ways. Transitive means... ?

Order

<is less than a < b
>is greater than a > b
≤ is less than or equal to (a < b) ∨ (a = b)
≥ is greater than or equal to (a > b) ∨ (a = b)

Transitive: ∀a,b,c (a < b ∧ b < c) ⟹ (a < c)

Order is less strict than equality. We only need it to be transitive.

Mapping

m(a) = b

A mapping replaces one thing with another. This example requires only one variable (unary) but a mapping might require two (binary) or more.

We will go into more detail about what a mapping is and how to construct one later. For now we just need to understand that we can apply a procedure to a value to get another value.

Formal Systems

  • Axioms: define the fundamentals of a system
  • Rules of Inference: govern how statements can be combined
  • Theorem: proven using axioms and rules
A formal system is a set of axioms and rules of inference that can be used to construct theorems. We would like our formal system to be consistent and complete. Consistent means no theorem can be proven both true and false from the axioms. Complete means that all theorems can be proven either true or false. Gödel proved that this is not possible with his Incompleteness Theorem.

Nothing But Set

We attempt to define a formal system in which the only objects are sets and the only thing a set can do is contain other sets.

Extensionality

∀a,b,c (c ∈ a ⟺ c ∈ b) ⟹ a = b

(a = b) ≝ ((a ⊆ b) ∧ (b ⊆ a))

Two sets that contain the same elements are the same set.

  • Reflexive: ∀a (a ⊆ a)
  • Symmetric: ∀a,b ((a ⊆ b) ∧ (b ⊆ a)) ⟺ ((b ⊆ a) ∧ (a ⊆ b)
  • Transitive: ∀a,b,c (a = b ∧ b = c) ⟹ (a = c)

Pairing

∀a,b ∃{a, b} ∀c (c∈{a, b} ⟺ (c = a ∨ c = b))

Any two sets can be used to create a third set which contains only those two sets as members. The two sets might be equal in which case we can create a set containing a single member.

Union

∀a ∃b ∀c∈b (∃d∈a c∈d)

Any set can be used to create a set by combining all the members of its members.

Given the set {{a, b}, {b, c}} the union of its members is: ∪{{a, b}, {b, c}} = {a, b, c}

Power Set

∀a ∃b ∀c (c∈b ⟺ c⊆a)

Any set can be used to create a set of its subsets.

For example, {a, b, c} has the following power set: {{}, {a}, {b}, {c}, {a, b}, {b, c}, {a, c}, {a, b, c}}

Regularity

∀a ∃b∈a ⟹ ∃c∈a ∄d (d∈a ∧ d∈c)

A set that contains any members must contain at least one member that does not share any members with itself. No set can contain itself.

Infinity

∃a (({}∈a) ∧ (∀b∈a b∪{b} ∈ a))

A set with infinitely many distinct members exists. Specifically the set containing the empty set and all sets we can get by taking the union of any member with a set containing only that member exists.

Specification (Schema)

∀a,p∃b ∀c c∈b ⟺ (c∈a ∧ P(c))

Given a set and a predicate there is another set which contains all members of the set for which the predicate is true. We can use this with any set and the predicate p(x): x ≠ x to create the empty set.

Replacement (Schema)

∀a,f∃b ∀c c∈b ⟺ ∃d∈a f(c) = d

Given a set and a mapping from that set to some other set there is a set which contains all the mapping results for all members of the original set.

Choice

∀a {}∉a ⟹ ∃f: a ⟶ ∪a ∀b∈a f(b) ∈ b

For any set that doesn't have any empty members a function exists that chooses one member from each of the sets members. While obvious for fininte sets, this is necessary for some infinite sets.

Addition ℕ

Let's define addition which maps two natural numbers to another natural number. We will represent it with a + sign between two natural numbers: a + b (Grassman 1861):

  • Base: ∀a∈ℕ a + 0 ≝ a
  • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)

Addition ℕ Example: 3 + 2

  • 3 + 2 = 3 + succ(1) :: Definition of 2
  • 3 + 2 = 3 + succ(succ(0)) :: Definition of 1
  • 3 + 2 = succ(3 + succ(0)) :: Step +
  • 3 + 2 = succ(succ(3 + 0)) :: Step +
  • 3 + 2 = succ(succ(3)) :: Base +
  • 3 + 2 = succ(4) :: Definition of 4
  • 3 + 2 = 5 :: Definition of 5
We can use this method to add any two natural numbers: break down the number on the left into successors, use the definition step to bring the successors outside, remove the zero and then evaluate the successors. To do this more efficiently we can memorize or use a lookup table.

Addition ℕ: Closed

∀a,b∈ℕ a + b ∈ℕ

  • Base: ∀a∈ℕ a + 0 ≝ a
  • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)

Addition is constructed entirely from successor operations. Since the successor of any natural number is also a natural number, addition is closed.

Addition ℕ: Identity

∀a∈ℕ a + 0 = a = 0 + a

  • 0 + 0 = 0 :: Base +
  • ∃a∈ℕ : 0 + a = a :: Assumption
  • 0 + succ(a) = succ(0 + a) :: Step +
  • 0 + succ(a) = succ(a) :: Induction

Addition ℕ: Associative

∀a,b,c∈ℕ (a + b) + c = a + (b + c)

  • (a + b) + 0 = a + b = a + (b + 0) :: Base +
  • ∃c∈ℕ ∀a,b∈ℕ : (a + b) + c = a + (b + c)
  • (a + b) + succ(c) = succ((a + b) + c) :: Step +
  • (a + b) + succ(c) = succ(a + (b + c)) :: Induction
  • (a + b) + succ(c) = a + succ(b + c) :: Step +
  • (a + b) + succ(c) = a + (b + succ(c)) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ a + b = b + a

We will prove two simpler results and make use of them to prove that addition is commutative

  • Successor + 1: ∀a∈ℕ succ(a) = a + 1
  • Commutative 1: ∀a∈ℕ a + 1 = 1 + a

Addition ℕ: Successor + 1

∀a∈ℕ succ(a) = a + 1

  • succ(a) = succ(a + 0) :: Base +
  • succ(a) = a + succ(0) :: Step +
  • succ(a) = a + 1 :: Definition of 1

Addition ℕ: Commutative 1

∀a∈ℕ a + 1 = 1 + a

  • 0 + 1 = 1 + 0 :: Identity +
  • ∃a∈ℕ : a + 1 = 1 + a :: Assumption
  • succ(a) + 1 = succ(a) + succ(0) :: Definition of 1
  • succ(a) + 1 = succ(succ(a) + 0) :: Step +
  • succ(a) + 1 = succ((a + 1) + 0) :: Successor + 1
  • succ(a) + 1 = succ(a + 1) :: Base +
  • succ(a) + 1 = succ(1 + a) :: Induction
  • succ(a) + 1 = 1 + succ(a) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ a + b = b + a

  • ∃b∈ℕ ∀a∈ℕ : a + b = b + a :: Assumption
  • a + succ(b) = a + (b + 1) :: Successor + 1
  • a + succ(b) = (a + b) + 1 :: Associativity
  • a + succ(b) = (b + a) + 1 :: Induction
  • a + succ(b) = b + (a + 1) :: Associativity
  • a + succ(b) = b + (1 + a) :: Communtative 1
  • a + succ(b) = (b + 1) + a :: Associativity
  • a + succ(b) = succ(b) + a :: Successor + 1

Addition ℕ

  • Definition:
    • Base: ∀a∈ℕ a + 0 ≝ a
    • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)
  • Closed: ∀a,b∈ℕ a + b ∈ℕ
  • Identity: ∀a∈ℕ a + 0 = a = 0 + a
  • Associative: ∀a,b,c∈ℕ (a + b) + c = a + (b + c)
  • Commutative: ∀a,b∈ℕ a + b = b + a

Multiplication ℕ

Let's define multiplication which maps two natural numbers to another natural number. We will represent it with a × sign between two natural numbers: a × b (Grassman 1861):

  • Base: ∀a∈ℕ a × 0 ≝ 0
  • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)

Multiplication ℕ Example: 3 × 2

  • 3 × 2 = 3 × succ(1) :: Definition of 2
  • 3 × 2 = 3 × succ(succ(0)) :: Definition of 1
  • 3 × 2 = 3 + (3 × succ(0)) :: Step ×
  • 3 × 2 = 3 + (3 + (3 × 0)) :: Step ×
  • 3 × 2 = 3 + (3 + 0) :: Base ×
  • 3 × 2 = 3 + 3 :: Addition
  • 3 × 2 = 6 :: Addition
We haven't shown that 3 plus 3 is 6 (the successor of 5). That's left as an exercise for the audience.

Multiplication ℕ: Closed

∀a,b∈ℕ a × b ∈ℕ

  • Base: ∀a∈ℕ a × 0 ≝ 0
  • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)

Multiplication is constructed entirely from successor operations and addition. Since the successor of any natural number is also a natural number and addition is closed over the natural numbers, multiplication is also closed.

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

Right identity:

  • a × 1 = a × succ(0) :: Definition of 1
  • a × 1 = a + (a × 0) :: Step ×
  • a × 1 = a + 0 :: Base ×
  • a × 1 = a :: Addition ℕ

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

Left identity:

  • 1 × 0 = 0 :: Base ×
  • ∃a∈ℕ 1 × a = a :: Assumption
  • 1 × succ(a) = 1 + (1 × a) :: Step ×
  • 1 × succ(a) = 1 + a :: Induction
  • 1 × succ(a) = succ(a) :: Addition ℕ

Multiplication ℕ: Distributive

∀a,b,c∈ℕ ((a + b) × c = (a × c) + (b × c)) ∧
(c × (a + b) = (c × a) + (c × b))

When we multiply a sum we can push the product through to each term. Because equality is symmetric we can also reverse this by pulling a common product out of each term.

Multiplication ℕ: Distributive

We separate this property into right and left distributive rules so we can prove them separately.

  • Right: ∀a,b,c∈ℕ ((a + b) × c = (a × c) + (b × c))
  • Left: ∀a,b,c∈ℕ (c × (a + b) = (c × a) + (c × b))

Multiplication ℕ: Distributive

∀a,b,c∈ℕ (a + b) × c = (a × c) + (b × c)

Consider the special case where c = 0:

  • (a + b) × 0 = 0 :: Base ×
  • (a + b) × 0 = 0 + 0 :: Addition ℕ
  • (a + b) × 0 = (a × 0) + (b × 0) :: Base ×

Multiplication ℕ: Distribute

  • (a + b) × 0 = (a × 0) + (b × 0)
  • ∃c∈ℕ ∀a,b∈ℕ : ((a + b) × c = (a × c) + (b × c)) :: Assumption
  • ((a + b) × succ(c) = (a + b) + ((a + b) × c) :: Step ×
  • ((a + b) × succ(c) = (a + b) + ((a × c) + (b × c)) :: Induction
  • ((a + b) × succ(c) = (a + (a × c)) + (b + (b × c)) :: Addition ℕ
  • ((a + b) × succ(c) = (a × succ(c)) + (b × succ(c)) :: Step ×

Multiplication ℕ: Distributive

∀a,b,c∈ℕ c × (a + b) = (c × a) + (c × b)

Consider the special case where b = 0:

  • c × (a + 0) = c × a :: Addition ℕ
  • c × (a + 0) = (c × a) + 0 :: Addition ℕ
  • c × (a + 0) = (c × a) + (c × 0) :: Definition Base

Multiplication ℕ: Distributive

  • c × (a + 0) = (c × a) + (c × 0)
  • ∃b∈ℕ ∀a,c∈ℕ : c × (a + b) = (c × a) + (c × b) :: Assumption
  • c × (a + succ(b)) = c × succ(a + b) :: Addition ℕ
  • c × (a + succ(b)) = c + (c × (a + b)) :: Step ×
  • c × (a + succ(b)) = c + ((c × a) + (c × b)) :: Induction
  • c × (a + succ(b)) = (c × a) + (c + (c × b)) :: Addition ℕ
  • c × (a + succ(b)) = (c × a) + (c × succ(b)) :: Step ×

Multiplication ℕ: Associative

∀a,b,c∈ℕ (a × b) × c = a × (b × c)

Consider the special case where c = 0:

  • (a × b) × 0 = 0 :: Base ×
  • (a × b) × 0 = a × 0 :: Base ×
  • (a × b) × 0 = a × (b × 0) :: Base ×

Multiplication ℕ: Associative

  • (a × b) × 0 = a × (b × 0)
  • ∃c∈ℕ ∀a,b∈ℕ : (a × b) × c = a × (b × c) :: Assumption
  • (a × b) × succ(c) = (a × b) + ((a × b) × c) :: Step ×
  • (a × b) × succ(c) = (a × b) + (a × (b × c)) :: Induction
  • (a × b) × succ(c) = a × (b + (b × c)) :: Distributive ×
  • (a × b) × succ(c) = a × (b × succ(c)) :: Step ×

Multiplication ℕ: Commutative

∀a,b∈ℕ a × b = b × a

Consider the special case where b = 0:

  • 0 × 0 = 0 :: Base ×
  • ∃a∈ℕ 0 × a = 0 :: Assumption
  • 0 × succ(a) = 0 + (0 × a) :: Step ×
  • 0 × succ(a) = 0 + 0 :: Induction
  • 0 × succ(a) = 0 :: Addition ℕ
  • 0 × succ(a) = succ(a) × 0 :: Base ×

Multiplication ℕ: Commutative

  • a × 0 = 0 × a
  • ∃b∈ℕ ∀a∈ℕ : a × b = b × a :: Assumption
  • a × succ(b) = a + (a × b) :: Step ×
  • a × succ(b) = a + (b × a) :: Induction
  • a × succ(b) = (b × a) + a :: Addition ℕ
  • a × succ(b) = (b × a) + (1 × a) :: Identity
  • a × succ(b) = (b + 1) × a :: Distributive
  • a × succ(b) = succ(b) × a :: Successor + 1

Multiplication ℕ

  • Definition:

    • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)
  • Closed: ∀a,b∈ℕ a × b ∈ℕ
  • Identity: ∀a∈ℕ a × 1 = a = 1 × a
  • Associative: ∀a,b,c∈ℕ (a × b) × c = a × (b × c)

Cardinality

Now that we have sets and natural numbers we can develop a way to talk about how many elements a set contains. We will need a few more tools.

Extensionality means that members of a set have no meaningful order.

∀a,b : {a, b} = {b, a}

Let's impose some order on them!

Ordered Pairs

Kuratowski 1921: ⟨a, b⟩ ≝ {{a}, {a, b}}

  • a is first in ⟨a, b⟩ ≝ ∀c∈⟨a, b⟩ : a∈c
  • b is second in ⟨a, b⟩ ≝ (∃c∈⟨a, b⟩ : b∈c) ∧ (∀c,d∈⟨a, b⟩ : c ≠ d ⟹ b∉c ∨ b∉d)
  • ∀a,b (a ≠ b) ⟺ (⟨a, b⟩ ≠ ⟨b, a⟩)
  • ∀a,b (a ≠ b) ⟺ {{a}, {a, b}} ≠ {{b}, {a, b}}

Relation

(r : D ⟶ R) ≝ { ⟨a, b⟩ : a∈D,b∈R }

A relation maps members of one set to another. Of course a relation is a set too.

  • D is the Domain of the Relation
  • R is the Range of the Relation
  • Application: r(a) = b ⟺ ⟨a, b⟩∈r

Function

(f : D ⟶ R) ≝ ∀a∈D (∃b∈R ⟨a, b⟩∈f) ∧
(∀b,c∈R (⟨a, b⟩∈f ∧ ⟨a, c⟩∈f) ⟹ b = c)

A function is a relation which maps every member of the domain to the exactly one member of the range.

Meet the Jections

Given function f : D ⟶ R

  • Injection: ∀a,b∈D f(a) = f(b) ⟹ a = b
    No two members of D are mapped to the same member of R
  • Surjection: ∀a∈R ∃b∈D f(b) = a
    Every member of R is mapped from at least one member of D
  • Bijection: both an injection and a surjection

Cardinality

If a bijection between a natural number and some other set then we say that other set has that many elements. For example, if a bijection exists between set a and the natural number 3 (which is the set {{}, {{}}, {{}, {{}}}}) we say that a has three elements.

Infinite Cardinality

If a bijection between a set b and the set of natural numbers itself exists then we say that b has a countably infinite number of elements.

Integers ℤ

ℤ ≝ { ∀a,b∈ℕ : ⟨a, b⟩ }

∀a,b∈ℕ [a - b] ≝ { ∀⟨c1, c2⟩∈ℤ : (c1 + b = c2 + a) }

Integers extend the natural numbers. They introduce inverses for addition and negative numbers which can represent concepts like debt.

Integers ℤ: Equality

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ

Rational number equality is determined by cross multiplying the members of the ordered pairs. It should be clear that this is reflexive since integer multiplication is commutative. Groups (G, ∘)

A group is a set G and a binary operation G ∘ G ⟶ G with four properties:

  • Closed: ∀a,b∈G : a ∘ b ∈G
  • Associative: ∀a,b,c∈G (a ∘ b) ∘ c = a ∘ (b ∘ c)
  • Identity: ∃e∈G ∀a∈G a ∘ e = a
  • Inverses: ∀a∈G ∃a-1∈G a ∘ a-1 = e

By definition identity and inverses work when applied from the right. Group axioms are enough to prove that these also work from the left.

  • e = a-1 ∘ (a-1)-1 :: Inverses
  • e = (a-1 ∘ e) ∘ (a-1)-1 :: Identity
  • e = (a-1 ∘ (a ∘ a-1)) ∘ (a-1)-1 :: Inverses
  • e = ((a-1 ∘ a) ∘ a-1) ∘ (a-1)-1 :: Associativity
  • e = (a-1 ∘ a) ∘ (a-1 ∘ (a-1)-1) :: Associativity
  • e = (a-1 ∘ a) ∘ e :: Inverses
  • e = (a-1 ∘ a) :: Identity

So inverses work regardless of which side they're applied. This means the identity also works from both sides:

  • e ∘ a = (a ∘ a-1) ∘ a :: Inverses
  • e ∘ a = a ∘ (a-1 ∘ a) :: Associativity
  • e ∘ a = a ∘ e :: Inverses
  • e ∘ a = a :: Identity

A group is the simplest mathematical structure that allows algebra. Suppose we have know a, b and
(x ∘ a = b):

  • (x ∘ a) ∘ a-1 = b ∘ a-1 :: Equality
  • x ∘ (a ∘ a-1) = b ∘ a-1 :: Associativity
  • x ∘ e = b ∘ a-1 :: Inverses
  • x = b ∘ a-1 :: Identity

An abelian group is a group with a commutative operation.

  • (G, ∘) is a group
  • Commutative: ∀a,b,c∈G (a ∘ b) = (b ∘ a)

Integers with addition form an abelian group: (ℤ, +)

Rings (R, ∘, ⊚)

A ring is a set and two binary operations R ∘ R ⟶ R and R ⊚ R ⟶ R with four properties:

  • (R, ∘) is an abelian group
  • Closed: ∀a,b∈R : a ⊚ b ∈ R
  • Associative: (a ⊚ b) ⊚c = a ⊚ (b ⊚ c)
  • Distributive: ∀a,b,c∈R
    (a ⊚ (b ∘ c) = (a ⊚ b) ∘ (a ⊚ c)) ∧
    ((b ∘ c) ⊚ a = (b ⊚ a) ∘ (c ⊚ a))
