Logic

Contrariwise, continued Tweedledee, if it was so, it might be; and if it were so, it would be; but as it isn't, it ain't. That's logic. — Through the Looking Glass by Lewis Carroll
Just in case Tweedledee's explanation isn't enough, let's go over a few things. Logic is a complex topic and addresses many concepts in the wider world. We only need a few relatively simple ideas here.

Predicate

A predicate is a statement that may be true or false based on the value of its variables.

  • Example: _____ is a fruit
  • An apple is a fruit (true)
  • A carrot is a fruit (false)
A predicate is a statement with one or more holes in it. Depending on what we put in the holes it may be true or false. If our predicate is blank is a fruit then it's true if we fill in the blank with apple but false if we use carrot instead.

Bound Variables

If P is a predicate with variable then the following are also predicates:

∀a P(a) P is true for all values of a
∃a P(a) P is true for some value of a
∄a P(a) P is false for all values of a
We can bind variables in predicates. The simplest ways to do this are the ones shown here. An upside down A asserts that the predicate is true for all values of a, sometimes with further qualitifaction. A backwards E asserts that some value for a exists for which the predicate is true. A backwards E with a line through it asserts that there is no value for a that makes the predicate true.

Compound Predicates

Given predicates P and Q the following are also predicates:

¬P not P (P is false)
(P ∧ Q) both P and Q are true
(P ∨ Q) at least one of P and Q are true
(P ⟹ Q) if P then Q (implication)
(P ⟺ Q) if and only if P then Q (biconditional)
We can combine simpler predicates into compound predicates in a few ways. We can negate a predicate by asserting that it is false. For example, if it is raining is a predicate then it is not raining is also a predicate. We can use logical and to assert that two predicates are both true. For example, we can say it is raining and the ground is wet. We can use logical or to assert that at least one of two predicates are true. For example, it is raining or the ground is wet. Finally we can combine predicates conditionally. For example, if it is raining then the ground is wet. This can be applied in both directions. For example, if and only if it is raining then the ground is wet.

Biconditional Laws

∀P,Q (P ⟺ Q) ≝ (P ⟹ Q) ∧ (Q ⟹ P)

Symmetric: ∀P,Q (P ⟺ Q) ⟹ (Q ⟺ P)

Transitive: ∀P,Q,R (P ⟺ Q) ∧ (Q ⟺ R) ⟹ (P ⟺ R)

Biconditional predicates can be swapped and transferred without changing results.

Liebniz's Law

∀a,b (a = b) ⟹ (P(a) ⟺ P(b))

When a and b are equal the value of any predicate using a is the same as the same predicate using b. We can substitute equal values without changing any results.

Deduction

Deduction applies general rules to reach specific conclusions.

  • Modus ponens: ((P ⟹ Q) ∧ P) ⟹ Q
  • Modus tollens: ((P ⟹ Q) ∧ ¬Q) ⟹ ¬P
  • Syllogism: ((P ⟹ Q) ∧ (Q ⟹ R)) ⟹ (P ⟹ R)
  • And so on...
Deduction allows us to derive specific conlusions from general rules. These are some deductive rules. Modus ponens allows us to prove consequences. So if we've proven both if it is raining then the ground is wet and it is raining then modus ponens proves that the ground is wet. Modus tollens is similar but instead lets us combine the same conditional with the ground is not wet to prove that it is not raining. Syllogism lets us combine conditionals in a transitive way. So given if it is raining the ground is wet and if the ground is wet the grass grows we can use syllogism to prove that if it is raining the grass grows.

Induction

Induction uses specific statements to prove more general conclusions. A base establishes a starting point and a step can be repeated indefinitely.

  • P(a) ∧ (P(b) ⟹ P(next(b)))
  • P(b) is true for anything that follows a
Induction allows us to build general rules from specific starting points. For example, suppose we have a ladder and we want to prove that we can climb to the top. We would need to show first that we can climb onto the bottom rung and second that from any rung we could reach the next one. We'll ingore practical constraints like getting tired or being afraid of heights.

Formal Systems

  • Rules of Inference: ways statements can be combined
  • Axioms: statements accepted as true
  • Theorem: statements proven as true using rules on axioms and other theorems
  • Definitions: short hand for concepts
We're going to build a formal system, though we're going to do it in an infromal and not entirely rigorous way. We know patience is a limited resource. A formal system is made up of four important parts. First, we have axioms. These are things we accept as true without proof. They are the foundation of the formal system. Second we have rules of inference which control what we're allowed to do when constructing a proof. Third we have theorems. These are statements that we prove using rules of inference with axioms and other theorems. Finally we have definitions which allow us to represent complex concepts in a more compact way.

Incompleteness

Desirable properties of a formal system:

  • Consistent: no statement can be proven both true and false (no paradoxes)
  • Complete: all statements can be proven either true or false (nothing missing)

Any formal system containing basic arithmetic is either inconsistent or incomplete.
(Gödel 1931 CE)

We would like our formal system to be both consistent and complete. Consistency means that no theorem can be proven both true and false. Complete means that all possible theorems can be proven one way or the other. Unfortunately, Kurt Gödel proved that no sufficiently expressive system can be both. So we will have to accept some limitations.

Nothing But Set

We will use set theory to construct relations, functions, numbers, arithmetic, algebra and geometry. Set theory is a formal system in which the only objects are sets and the only thing sets can do is contain other objects. (Zermelo-Fraenkel 1908-1922 CE)

Axiomatic set theory is the foundation of this course. So far we've covered the most important rules of inference. This seciton will explain the axioms of our formal system. The axioms described here were first developed by Ernst Zermelo, Abraham Fraenkel and Thorlaf Skolem. Alternative formulations, such as Von Neumann's Category Theory, also exist but we won't use them here.

Set Predicates

a ∈ {a, b, c} member of
a ∉ {b, c} ⟹
(b≠a ∧ c≠a) 		not a member of
(a ∉ b) ≝ ¬(a ∈ b)
{a} ⊆ {a, b}
{a, b} ⊆ {a, b} 		subset of
(a ⊆ b) ≝ ∀c∈a c∈b
We use a symbol that resembles a lower case e to indicate that something is a member of a set. We'll put a line through it as a compact way to invert the predicate. We'll use a sideways letter U with a line under it to indicate that one set is a subset of a second set, which means all members of the first set are also members of the second. Notice that every set is a subset of itself.

Set Contents

We describe sets using one of these forms.

{ a, b, c } Enumeration
{ a∈b : P(a) } Specification (filtered by predicate)
{ f(a) : a∈b } Replacement (function outputs)
We usually describe sets in one of these two ways. When a set contains only a few selected members, we enumerate them, separated by commas. Larger sets are difficult to represent this way so we describe them using a predicate which selects members from some other set.

Empty Set

∃{} ∀a a∉{}

The empty set is a set that doesn't contain anything.

Extensionality Same members, same set
Pairing Make a set with two members
Union Sets can be merged
Power Set Sets of subsets exist
Regularity Sets can't contain themselves
Specification A subset exists for any predicate
Replacement A set exists for any function
Infinity There is at least one infinite set
Choice One member from each set
These nine axioms are the foundation of Zermelo-Fraenkel set theory plus the axiom of choice. This is sometimes referred to as ZFC. We'll describe each axiom in detail.

Extensionality

∀a,b,c (c ∈ a ⟺ c ∈ b) ⟹
(a ⊆ b) ∧ (b ⊆ a) ⟹ (a = b)

Two sets that contain the same elements are the same set.

Extensionality means that any two sets that contain the same elements are the same set.

Pairing

∀a,b ∃{a, b} ∀c (c∈{a, b} ⟺ (c = a ∨ c = b))

Any two sets can be used to create a third set which contains only those two sets as members.

Pairing allows us to use any two sets to create a third set with only those two sets as members. We can use the same set twice as members to create a set with only one member.

Union

∀a ∃∪a ∀b∈∪a (∃c∈a b∈c)

Any set can be used to create a set by combining all the members of its members.

∪{{a, b}, {b, c}} = {a, b, c}

Union allows us to combine the members of the members of a set. We can use this along with the axiom of pairing to create a union operator which combines two sets into one.

Power Set

∀a ∃℘a ∀b (b∈℘a ⟺ b⊆a)

Any set can be used to create a set of its subsets.

℘{a, b, c} =
{{}, {a}, {b}, {c}, {a, b}, {b, c}, {a, c}, {a, b, c}}

Every set has a power set, which is the set of all subsets of the original set.

Regularity

∀a ∃b∈a ⟹ ∃c∈a ∄d (d∈a ∧ d∈c)

A set that contains any members must contain at least one member that does not share any members with itself. No set can contain itself.

Regularity prevents sets from containing themselves. The form given here prevents chains of sets that eventually come back to themselves and other such tricks. This axiom prevents paradoxes that afflicted earlier set theories.

Specification (Schema)

∀a,P ∃b ∀c c∈b ⟺ (c∈a ∧ P(c))

Given a set and a predicate there is another set which contains all members of the set for which the predicate is true. This allows us to filter sets.

Specification is actually an axiom schema because there is technically one axiom for every possible predicate. This schema allows us to use any predicate to filter the elmeents of any set.

Replacement (Schema)

∀a,f ∃b ∀c c∈b ⟺ ∃d∈a f(c) = d

Given a set and a function from that set to some other set there is a set which contains all the function outputs for all members of the set.

Replacement is another schema but this time it allows us to use a function to create one set from another. There is one axiom for each possible mapping. We haven't yet explained what a function is or how to construct one. We will do that once we've established a few important tools.

Infinity

∃a (({}∈a) ∧ (∀b∈a ∪{b, {b, b}} ∈ a))

A set with infinitely many distinct members exists. Specifically the set containing the empty set and all sets we can get by taking the union of any member with a set containing only that member exists.

The axiom of infinity guarantees that at least one infinite set exists. This will allow us to create and work with other infinite sets.

Choice

∀a {}∉a ⟹ ∃f: a ⟶ ∪a ∀b∈a f(b) ∈ b

For any set that doesn't have any empty members a function exists that chooses one member from each of the sets members.

The axiom of choice has been controversial and it allows some unintuitive results such as the Banach-Tarski theorem. While not necessary for fininte collections of sets, this is useful for working with infinite sets.

Relations

A relation describes, paritions or orders a set. Of course a relation is also a set.

Extensionality means that members of a set have no meaningful order.

∀a,b : {a, b} = {b, a}

Let's impose some order on them!

Ordered Pairs

⟨a, b⟩ ≝ {{a}, {a, b}} (Kuratowski 1921 CE)

    • ∀a,b (a ≠ b) ⟺ (⟨a, b⟩ ≠ ⟨b, a⟩)
    • ∀a,b (a ≠ b) ⟺ {{a}, {a, b}} ≠ {{b}, {a, b}}
  • a is first in ⟨a, b⟩ ≝ ∀c∈⟨a, b⟩ a∈c
  • b is second in ⟨a, b⟩ ≝ (∃c∈⟨a, b⟩ b∈c) ∧
    (∀c,d∈⟨a, b⟩ c ≠ d ⟹ b∉c ∨ b∉d)
  • ∪⟨a, b⟩ = ∪{{a}, {a, b}} = {a, b}

Cartesian Product

∀a,b (a × b) ≝
{ ⟨a', b'⟩ ∈ ℘℘(a ∪ b) : a'∈a ∧ b'∈b }

Pairing, union and power set and specification can be used to create a set product.

{a, b, c} × {c, d} =
{ ⟨a, c⟩, ⟨a, d⟩, ⟨b, c⟩, ⟨b, d⟩, ⟨c, c⟩, ⟨c, d⟩ }

Relation

a R b ≝ ⟨a, b⟩∈R

A relation is a set of ordered pairs.

  • ∪∪R contains all members
  • Domain(R) ≝ { a∈∪∪R : ∃b∈∪∪R ⟨a, b⟩∈R }
  • Range(R) ≝ { b∈∪∪R : ∃a∈∪∪R ⟨a, b⟩∈R }
  • R ⊆ Domain(R) × Range(R)

Relation Properties

  • Reflexive: ∀a∈∪∪R (a R a)
  • Symmetric: ∀a,b∈∪∪R (a R b) ⟺ (b R a)
  • Antisymmetric:
    ∀a,b∈∪∪R (a R b) ∧ (b R a) ⟹ (a = b)
  • Transitive: ∀a,b,c∈∪∪R (a R b) ∧ (b R c) ⟺ (a R c)
  • Connex: ∀a,b∈∪∪R (a R b) ∨ (b R a)
These are some properties a relation might have. We will use these as building blocks for higher level concepts.

Equivalence

An equivalence relation has three properties:

  • Reflexive: ∀a (a ≑ a)
  • Symmetric: ∀a,b (a ≑ b) ⟺ (b ≑ a)
  • Transitive: ∀a,b,c (a ≑ b ∧ b ≑ c) ⟹ (a ≑ c)

Equivalence relations partition sets into equivalence classes.

An equivalence relation has three special properties. Reflexive means every member of the domain relates to itself. Symmetric means we can always reverse the members. Transitive means that whenever there is a chain we can take a shortcut.

Partial Order

A partial order is a relation build on an equivalence relation with the following properties:

  • Reflexive: ∀a (a ⋖ a)
  • Antisymmetric: ∀a,b (a ⋖ b) ∧ (b ⋖ a) ⟹ (a = b)
  • Transitive: ∀a,b,c (a ⋖ b) ∧ (b ⋖ c) ⟹ (a ⋖ c)
A partial order is a relation on a set that that makes some but not necessarily all pairs of elements comparable. To be a partial order, the relation must be reflexive, antisymmetric and transitive.

Total Order

A total order is a partial order with connex:

  • Reflexive, Antisymmetric and Transitive
  • Connex: ∀a,b (a ⋖ b) ∨ (b ⋖ a)
A total order is a partial order with the connex property, which means that every pair of elements is related in one direction or the other.

Equality: Reflexive

∀a a = a

Every set is equal to itself. Obviously?

Equality: Symmetric

∀a,b (a = b) ⟺ (b = a)

  • (a = b) ⟹ (P(a) ⟺ P(b)) :: Leibniz's Law
  • (a = b) ⟹ (P(b) ⟺ P(b)) :: Symmetric Biconditional
  • (a = b) ⟹ (b = a) :: Leibniz's Law

Equality: Transitive

∀a,b,c (a = b) ∧ (b = c) ⟹ (a = c)

  • (a = b) ∧ (b = c) ⟹ (P(a) ⟺ P(b)) :: Leibniz's Law
  • (a = b) ∧ (b = c) ⟹ (P(a) ⟺ P(b)) ∧ (P(b) ⟺ P(c)) :: Leibniz's Law
  • (a = b) ∧ (b = c) ⟹ (P(a) ⟺ P(c)) :: Transitive Biconditional
  • (a = b) ∧ (b = c) ⟹ (a = c) :: Leibniz's Law

Equality: Equivalence

∀a ∃{ ⟨b, c⟩ : b∈a ∧ c∈a ∧ b = c }

Set equality can be used to construct an equivalence relation over any set: this is always reflexive, symmetric and transitive.

Subset: Reflexive

∀a a ⊆ a

Every set is a subset of itself by definition, so the subset operator is reflexive.

Subset: Antisymmetric

∀a,b (a ⊆ b) ∧ (b ⊆ a) ⟹ (a = b)

Two sets that are each subsets of the other must contain the same elements. Extensionality makes them equal.

Subset: Transitive

∀a,b,c (a ⊆ b) ∧ (b ⊆ c) ⟹ (a ⊆ c)

  • (a ⊆ b) ∧ (b ⊆ c) ⟹ ∀d (d∈a ⟹ d∈b)
    :: Definition Subset
  • (a ⊆ b) ∧ (b ⊆ c) ⟹ ∀d (d∈b ⟹ d∈c)
    :: Definition Subset
  • (a ⊆ b) ∧ (b ⊆ c) ⟹ (a ⊆ c) :: Definition Subset

Subsets: Partial Order

∀a ∃{ ⟨b, c⟩ : b∈a ∧ c∈a ∧ b ⊆ c }

Subset relationships can be used to construct a partial order over any set. This is always reflexive, antisymmetric and transitive. It's not always connex, so subset is not always a total order:

{{a, b}, {a}, {a, c}}

Equality ℕ

Every natural number is a distinct set so set equality works for testing natural number equality. This is reflexive, symmetric and transitive because set equality has these properites.

Order ℕ

∀a,b∈ℕ a ≤ b ≝ a ∈ succ(b)
∀a,b∈ℕ a ≥ b ≝ b ∈ succ(a)
∀a,b∈ℕ a < b ≝ ¬(a ≥ b) ⟺ (a ∈ b)
∀a,b∈ℕ a > b ≝ ¬(a ≤ b) ⟺ (b ∈ a)

Natural number order is built on set membership.

Order ℕ: Reflexive

Natural numbers are always contained within their successors, so order is reflexive.

  • ∀a∈ℕ a∈succ(a)
  • ∀a∈ℕ a∈∪{a, {a, a}}
  • ∀a∈ℕ a ≤ a
Natural number order is reflexive because every natural number is contained in its own successor. This is a clear consequence of the union operator.

Order ℕ: Antisymmetric

Every natural number is a subset of all of its successors. This makes natural number order antisymmetric due to regularity.

  • ∀a∈ℕ a ∉ a ⟹ succ(a) ∉ a
  • ∀a,b∈ℕ (a ∈ succ(b)) ∧ (b ∈ succ(a)) ⟹ (a = b)
  • ∀a,b∈ℕ (a ≤ b) ∧ (b ≤ a) ⟹ (a = b)

Order ℕ: Transitive

Every natural number contains all predecessors. This means each contains not only its own predecessors but also the predecessors of those predecessors.

  • ∀a,b,c∈ℕ (a∈succ(b)) ∧ (b∈succ(c)) ⟹ (a∈succ(c))
  • ∀a,b,c∈ℕ (a ≤ b) ∧ (b ≤ c) ⟹ (a ≤ c)
Every natural number is contained in its successor and indeed in the successor of its successor. So if a is less than b and b is less than c, then a is contained in b and b is contained in c. Since a is contained in b it's contained in all of b's successors, including c.

Order ℕ: Connex

Every pair of natural numbers has an order. Either the first is less than or equal to the second, the second is less than or equal to the first or both.

  • ∀a,b∈ℕ (a∈succ(b)) ∨ (b∈succ(a))
  • ∀a,b∈ℕ (a ≤ b) ∨ (b ≤ a)

Natural Numbers ℕ

  • succ(a) ≝ ∪{a, {a, a}} = a ∪ {a}
  • ℕ ≝ { a : (a = {}) ∨ ∃b∈ℕ a = succ(b) }
  • Equality: Set Equality
  • Order: ∀a,b∈ℕ a ≤ b ≝ a ∈ succ(b)

Addition ℕ

Let's define addition which maps two natural numbers to another natural number. We will represent it with a + sign between two natural numbers: a + b (Grassman 1861 CE):

  • Base: ∀a∈ℕ a + 0 ≝ a
  • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)

Addition ℕ Example: 3 + 2

  • 3 + 2 = 3 + succ(1) :: Definition of 2
  • 3 + 2 = 3 + succ(succ(0)) :: Definition of 1
  • 3 + 2 = succ(3 + succ(0)) :: Step +
  • 3 + 2 = succ(succ(3 + 0)) :: Step +
  • 3 + 2 = succ(succ(3)) :: Base +
  • 3 + 2 = succ(4) :: Definition of 4
  • 3 + 2 = 5 :: Definition of 5
We can use this method to add any two natural numbers: break down the number on the left into successors, use the definition step to bring the successors outside, remove the zero and then evaluate the successors. To do this more efficiently we can memorize or use a lookup table.

Addition ℕ: Closed

∀a,b∈ℕ a + b ∈ℕ

  • Base: ∀a∈ℕ a + 0 ≝ a
  • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)

Addition is constructed entirely from successor operations. Since the successor of any natural number is also a natural number, addition is closed.

Addition ℕ: Identity

∀a∈ℕ a + 0 = a = 0 + a

  • 0 + 0 = 0 :: Base +
  • ∃a∈ℕ : 0 + a = a :: Assumption
  • 0 + succ(a) = succ(0 + a) :: Step +
  • 0 + succ(a) = succ(a) :: Induction

Addition ℕ: Associative

∀a,b,c∈ℕ (a + b) + c = a + (b + c)

  • (a + b) + 0 = a + b = a + (b + 0) :: Base +
  • ∃c∈ℕ ∀a,b∈ℕ : (a + b) + c = a + (b + c)
  • (a + b) + succ(c) = succ((a + b) + c) :: Step +
  • (a + b) + succ(c) = succ(a + (b + c)) :: Induction
  • (a + b) + succ(c) = a + succ(b + c) :: Step +
  • (a + b) + succ(c) = a + (b + succ(c)) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ a + b = b + a

We will prove two simpler results and make use of them to prove that addition is commutative

  • Commutative 1: ∀a∈ℕ a + 1 = 1 + a

∀a∈ℕ succ(a) = a + 1

Multiplication ℕ

Let's define multiplication which maps two natural numbers to another natural number. We will represent it with a × sign between two natural numbers: a × b (Grassman 1861 CE):

  • Base: ∀a∈ℕ a × 0 ≝ 0
  • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)

Multiplication ℕ Example: 3 × 2

  • 3 × 2 = 3 × succ(1) :: Definition of 2
  • 3 × 2 = 3 × succ(succ(0)) :: Definition of 1
  • 3 × 2 = 3 + (3 × succ(0)) :: Step ×

  • 3 × 2 = 3 + (3 + 0) :: Base ×
  • 3 × 2 = 3 + 3 :: Addition
  • 3 × 2 = 6 :: Addition
We haven't shown that 3 plus 3 is 6 (the successor of 5). That's left as an exercise for the audience.

Multiplication ℕ: Closed

∀a,b∈ℕ a × b ∈ℕ

  • Base: ∀a∈ℕ a × 0 ≝ 0
  • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)

Multiplication is constructed entirely from successor operations and addition. Since the successor of any natural number is also a natural number and addition is closed over the natural numbers, multiplication is also closed.

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

  • a × 1 = a × succ(0) :: Definition of 1
  • a × 1 = a + (a × 0) :: Step ×
  • a × 1 = a + 0 :: Base ×
  • a × 1 = a :: Addition ℕ

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

Left identity:

  • 1 × 0 = 0 :: Base ×

  • 1 × succ(a) = succ(a) :: Addition ℕ

∀a,b,c∈ℕ ((a + b) × c = (a × c) + (b × c)) ∧

Integers extend the natural numbers. They introduce inverses for addition and negative numbers which can represent concepts like debt.

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ

    (a1 + b2 = b1 + a2) ∧ (b1 + c2 = c1 + b2) ⟺
    (a1 + c2 = a2 + c1)
Integer equality means that the gap between the numbers has the same number steps in the same direction.

∀a,b∈ℕ ⟨a, b⟩ = ⟨succ(a), succ(b)⟩

Every integer has a simplest form, which has at least one member that is natural number zero.

Rational numbers extend the integers. They introduce inverses for multiplication and can represent fractional quantities.

Rational Numbers ℚ: Equality

∀⟨a1, a2⟩∈ℚ, ⟨b1, b2⟩∈ℚ
(⟨a1,a2⟩ =ℚ ⟨b1,b2⟩) ≝ (a1 × b2 = a2 × b1)

  • Reflexive: ∀⟨a1, a2⟩∈ℚ a1 × a2 = a2 × a1
  • Symmetric: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℚ
    a1 × b2 = a2 × b1 ⟺ b1 × a2 = b2 × a1
  • Transitive: ∀⟨a1, a2⟩,⟨b1, b2⟩,⟨c1, c2⟩∈ℚ
    (a1 × b2 = a2 × b1) ∧ (b1 × c2 = b2 × c1) ⟺
    (a1 × c2 = a2 × c1)
Rational number equality is determined by cross multiplying the members of the ordered pairs. It should be clear that this is reflexive since integer multiplication is commutative. In the same way, we can swap the equality and use commutative integer multiplication to show that this is symmetric. Transitivity is more complex but hopefully it's clear enough.

Let's define addition which maps two rational numbers to another. This will preserve all properties of integer addition, including inverses. As with integers we will represent this with a + sign between two rational numbers: a + b

∀a,b∈ℚ a + b ≝ ⟨a1, a2⟩ + ⟨b1, b2⟩ =

∀ a,b∈ℤ ⟨a, 1⟩ + ⟨b, 1⟩ =
⟨(a × 1) + (1 × b), 1 × 1⟩ = ⟨a + b, 1⟩

Adding any two rational numbers that match integers results in the integer their sum would match.

Addition ℚ: Closed

∀a,b∈ℚ a + b ∈ℚ

∀a,b∈ℚ a + b ≝ ⟨a1, a2⟩ + ⟨b1, b2⟩ =
⟨(a1 × b2) + (a2 × b1), a2 × b2⟩

Addition for rational numbers is constructed entirely from integer arithmetic, which is closed. The product of two positive numbers is always positve so second element is positive.

We have to be a bit more careful here because not all ordered pairs of integers are rational numbers. The second member must be greater than zero.

Multiplication ℚ

Let's define multiplication which maps two rational numbers to another. This will presearve all properties of integer multiplication and will add inverses for everything except zero. As with integers we will represent this with a × sign between two rational numbers: a × b

∀a,b∈ℚ a × b ≝ ⟨a1, a2⟩ × ⟨b1, b2⟩ = ⟨a1 × b1, a2 × b2⟩

Multiplication ℚ Example: 3 × 5/2

  • 3ℚ × 5/2ℚ = ⟨3, 1⟩ × ⟨5, 2⟩
  • 3ℚ × 5/2ℚ = ⟨3 × 5, 1 × 2⟩
  • 3ℚ × 5/2ℚ = ⟨15, 2⟩
  • 3ℚ × 5/2ℚ = 15/2ℚ

Multiplication ℚ: Isomorphic

∀ a,b∈ℤ ⟨a, 1⟩ × ⟨b, 1⟩ = ⟨a × b, 1 × 1⟩ = ⟨a × b, 1⟩

Multiplying any two rational numbers that match integers results in the integer their product would match.

Multiplication ℚ: Closed

∀a,b∈ℚ a × b ∈ℚ

∀a,b∈ℚ a × b ≝ ⟨a1, a2⟩ × ⟨b1, b2⟩ = ⟨a1 × b1, a2 × b2⟩

Because integer multiplication is commutative and two positive integers always multiply to another positive, rational number multiplication is closed.

Multiplication ℚ: Identity

∀a∈ℚ a × 1 = a = 1 × a

  • ⟨a1, a2⟩ × ⟨1, 1⟩ = ⟨a1 × 1, a2 × 1⟩ :: Definition ×
  • ⟨a1, a2⟩ × ⟨1, 1⟩ = ⟨a1, a2⟩ :: Multiplication ℤ
  • ⟨a1, a2⟩ × ⟨1, 1⟩ = ⟨1 × a1, 1 × a2⟩ :: Multiplication ℤ
  • ⟨a1, a2⟩ × ⟨1, 1⟩ = ⟨1, 1⟩ × ⟨a1, a2⟩ :: Definition ×
Multiplying a rational number by one doesn't change it.

Multiplication ℚ: Associative

∀a,b,c∈ℚ (a × b) × c = a × (b × c)

  • (⟨a1, a2⟩ × ⟨b1, b2⟩) × ⟨c1, c2⟩ =
    ⟨(a1 × b1) × c1, (a2 × b2) × c2⟩ :: Definition ×
  • (⟨a1, a2⟩ × ⟨b1, b2⟩) × ⟨c1, c2⟩ =

∀a,b∈ℚ a × b = b × a

Real Numbers ℝ

ℝ ≝ { ⟨L,U⟩ : L,U⊆ℚ ∧ dedekind(L, U) }

Real numbers extend the rational numbers. They make it possible to find roots of many algebraic expressions as well as represent some convergant infinite series. (Dedekind 1858-1862)

Dedekind Cuts

∀L,U⊆ℚ dedekind(L, U) ≝

  • ∃a∈ℚ a∈L
  • ∀a,b∈ℚ (a ∈ L) ∧
    (b < a) ⟹ (b ∈ L)
  • ∀a,b∈ℚ (b ∉ U) ∧
    (a < b) ⟹ (a ∈ L)
  • ∀a,b∈ℚ (a ∈ L) ⟹
    ∃c∈ℚ (c>a) ∧ (c∈L)
  • ∃b∈ℚ b∈U
  • ∀a,b∈ℚ (b ∈ U) ∧
    (a > b) ⟹ (a ∈ U)
  • ∀a,b∈ℚ (a ∉ L) ∧
    (b > a) ⟹ (b ∈ U)
  • ∀a,b∈ℚ (b ∈ U) ⟹
    ∃c∈ℚ (c<b) ∧ c∈U

Real Numbers ℝ Example

0ℝ ≝ ⟨{a∈ℚ : a < 0ℚ}, {b∈ℚ : b > 0ℚ}⟩

Real number 0 is constructed by separating rational numbers less than 0 and greater than 0.

Real Numbers ℝ Example

∀a∈ℚ ∃ ⟨{b∈ℚ : b < a}, {c∈ℚ : c > a}⟩

All rational numbers can be used to construct a real number.

Real Numbers ℝ Example

This is the other.

Real Numbers ℝ Irrational

∀⟨L,U⟩∈ℝ (L ∪ U = ℚ) ⟹ ⟨L,U⟩ is irrational

An irrational number is a real number that does not correspond to any rational number. This is true when the union of the upper and lower set is equal to the entire set of rational numbers.

Real Numbers ℝ Equality

Real numbers are equal if and only if the sets that represent them are equal. This is reflexive symmetric and transitive because set equality has these properties.

Real Numbers ℝ Order

(⟨AL,AU⟩ ≤ ⟨BL,BU⟩) ≝ (AL ⊆ BL) ∧ (BU ⊆ AU)

A real number is less than or equal to another if its lower set is a subset of the others lower set. This is reflexive, antisymmetric, transitive and connex because the subset operator is all of these things.

  • ℝ ≝ { ⟨L,U⟩ : L,U⊆ℚ ∧ dedekind(L, U) }
  • Equality: Set Equality
  • Order: (⟨AL,AU⟩ ≤ ⟨BL,BU⟩) ≝ (AL ⊆ BL) ∧ (BU ⊆ AU)

Group ⟬G, ∘⟭

A group is a set and a binary function G ∘ G ⟶ G:

  • Closed: ∀a,b∈G : a ∘ b ∈G
  • Associative: ∀a,b,c∈G (a ∘ b) ∘ c = a ∘ (b ∘ c)
  • Identity: ∃e∘∈G ∀a∈G a ∘ e∘ = a
  • Inverses: ∀a∈G ∃a-1∈G a ∘ a-1 = e∘

A group is the simplest mathematical structure that makes algebra possible.

Abstraction

By definition identity and inverses work when applied from the right.

  • Identity: ∃e∘∈G ∀a∈G a ∘ e∘ = a
  • Inverses: ∀a∈G ∃a-1∈G a ∘ a-1 = e∘

Do these work from the left?

  • ∃e∘∈G ∀a∈G e∘ ∘ a ≟ a
  • ∀a∈G ∃a-1∈G a-1 ∘ a ≟ e∘

Proof: Commutative Inverses

  • e∘ = a-1 ∘ (a-1)-1 :: Inverses
  • e∘ = (a-1 ∘ e∘) ∘ (a-1)-1 :: Identity
  • e∘ = (a-1 ∘ (a ∘ a-1)) ∘ (a-1)-1 :: Inverses
  • e∘ = ((a-1 ∘ a) ∘ a-1) ∘ (a-1)-1 :: Associativity
  • e∘ = (a-1 ∘ a) ∘ (a-1 ∘ (a-1)-1) :: Associativity
  • e∘ = (a-1 ∘ a) ∘ e∘ :: Inverses
  • e∘ = a-1 ∘ a :: Identity

Proof: Commutative Identity

  • e∘ ∘ a = (a ∘ a-1) ∘ a :: Inverses
  • e∘ ∘ a = a ∘ (a-1 ∘ a) :: Associativity
  • e∘ ∘ a = a ∘ e∘ :: Inverses
  • e∘ ∘ a = a :: Identity

Equation Solving

Suppose we have know a, b and (a ∘ x = b):

  • a-1 ∘ (a ∘ x) = a-1 ∘ b :: Equality
  • (a-1 ∘ a) ∘ x = a-1 ∘ b :: Associativity
  • e∘ ∘ x = a-1 ∘ b :: Inverses
  • x = a-1 ∘ b :: Identity

Abelian Group ⟬G, ∘⟭

An abelian group has a commutative function:

  • ⟬G, ∘⟭ is a group
  • Commutative: ∀a,b,c∈G (a ∘ b) = (b ∘ a)

Integers with addition are an abelian group: ⟬ℤ, +⟭

Ring ⟬R, ∘, ⊚⟭

A ring is a set and two binary functions R ∘ R ⟶ R and R ⊚ R ⟶ R with four properties:

  • ⟬R, ∘⟭ is an abelian group
  • Closed: ∀a,b∈R : a ⊚ b ∈ R
  • Associative: (a ⊚ b) ⊚c = a ⊚ (b ⊚ c)
  • Distributive: ∀a,b,c∈R
    (a ⊚ (b ∘ c) = (a ⊚ b) ∘ (a ⊚ c)) ∧
    ((b ∘ c) ⊚ a = (b ⊚ a) ∘ (c ⊚ a))

  • Commutative: ∀a,b,c∈R (a ⊚ b) = (b ⊚ a)

Integers with addition and multiplication form a commutative ring ⟬ℤ, +, ×⟭

Integers with addition and multiplication form a unitary ring ⟬ℤ, +, ×⟭

  • ∀a∈ℝ ∀i<0∈ℤ ⟨a, ⟨i, ⟨i, t⟩⟩⟩ =𝕄 ⟨-a, t⟩

Equal integers can be combined. Unequal integers can be swapped by negating the real number.

These are the ways we can transform the components while preserving equality. When the tuples of both components are empty, component equality is the same as real number equality for the first element. The second rule shows that we can prepend the same integer to both tuples without changing equality. The third and fourth rules allow us to collapse adjacent pairs of integers in the tuple. If these integers are negative we must flip the sign of the real number. Finally, the last rule allows us to swap pairs of integers as long as we flip the sign of the real number.

A multivector is a tuple consisting of one or more multivector components.

A multivector can be constructed in three ways. One is to convert a real number by making it the first member of an ordered pair and using the empty set as the second. Another is to take an existing ordered pair multivector and prepend an integer to the tuple. And finally we can create the sum of any two multivectors.

Multivectors the same if their components are the same. Swapping is allowed and components with matching tuples can be merged.

Multivector equality follows these rules. Any components with a zero value real number can be dropped. If the tuples of two adjacent components are the same they can be combined into a single component. If the first component of two multivectors have component equality and the rest of the multivector is the same, then the multivectors are equal. Two adjacent components of a multivector can be swapped without changing equality.

Norm 𝔾

If the product of a multivector and its conjugate is a scalar then this scalar is the norm squared. Otherwise there is no norm.

  • M M* = |M|2
  • |M| = +√(M M*)

Inverse 𝔾

If the product of the multivector and its conjugate is a non-zero scalar then the inverse is the multivector times the multiplicative inverse of that scalar (which is the norm squared).

  • M-1 = M (M M*)-1
  • M M-1 = M-1 M = 1
