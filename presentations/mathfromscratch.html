Math From Scratch

If you want to make an apple pie from scratch, you must first create the universe. â€” Carl Sagan
Creating the universe is rather a tall order. We'll settle for creating all of mathematics. We'll do that by stealing the best ideas invented by smart mathematicians over thousands of years.

Mathematics has no generally accepted definition. We could describe it as a collection of tools for understanding patterns and for studying structure and change. People have studied mathematics for centuries and found applications for it in every field of science and technology.

This is an attempt to build the most important parts of math from the simplest possible foundations. Forget everything you know about math and let's begin at the beginning.

For the purpose of this presentation, forget everything you know about math.

Foundation

We will need a few ideas to build upon.

  • Predicates
  • Notation
  • Induction

Predicates

A predicate is a statement that may be true or false based on the value of its variables.

Notation

We will use symbols to make statements more compact.

=is equal to

We will use these symbols to compare expressions. Equals means the expressions on both sides are the same. Not equals means they're different. If there is some notion of order, the less than and related symbols describe how the expressions on the left and right relate.

These symbols express logical ideas. The upside down A means that the expression to follow is true for all qualified things. The backwards E means that there is some qualified thing for which the epression is true. Backwards E with a slash means there isn't. And means the expressions on both sides are true. Or means at least one of them is true. Implies means that if the expression on the left is true then the one on the right is too. Equivalence is a kind of double sided implies; the expression on the right is true whenever the one on the left is, and vice versa.
∈is a member of
∉is not a member of
∪union
∩intersection
⊆is a subset of
⊂is a proper subset of
These symbols are used describe relationships for sets. The member symbol means the expression on the left is contained in the set described on the right. The U symbol refers to the combination of elements from sets on both sides. The upside-down U refers to the set of elements that appear in both sets on either side. Subset means that all members of the set on the left are also part of the set on the right. Proper subset is the same, except that there is at least one member of the set on the right that isn't in the set on the left.

Induction

An inductive proof is one that has a base case and a step. The base case shows that the predicate is true for some value and the step shows that if the predicate is true for some value then it is also true for a success of that value.

The only objects we will consider are sets. The only thing a set can do is contain things as members. We will use eight axioms defined by Zermelo and Fraenkel between 1908 to 1921, plus one more added later, as a foundation.

Extensionality

∀a,b : ∀c (c ∈ a ⟺ c ∈ b) ⟹ a = b

Two sets that contain the same elements are the same set.

  • Reflexive: ∀a : a = a
  • Symmetric: ∀a,b : (a = b) ⟺ (b = a)
  • Transitive: ∀a,b,c : (a=c ∧ b=c) ⟹ (a = b)

Pairing

∀a,b ∃{a, b} ∀c (c∈{a, b} : ⟹ (c = a ∨ c = b))

Any two sets can be used to create a third set which contains only those two sets as members. The two sets might be equal in which case we can create a set containing a single member.

Union

∀a ∃b : ∀c∈b (∃d∈a : c∈d)

Any set can be used to create a set by combining all the members of its members.

Given the set {{a, b}, {b, c}} the union of its members is:

∪{{a, b}, {b, c}} = {a, b, c}

Power Set

∀a ∃b ∀c (c∈b ⟺ c⊆a)

Any set can be used to create a set of its subsets.

For example, {a, b, c} has the following power set: {{}, {a}, {b}, {c}, {a, b}, {b, c}, {a, c}, {a, b, c}}

Regularity

∀a ∃b∈a ⟹ ∃c∈a : ∄d (d∈a ∧ d∈c)

A set that contains any members must contain at least one member that does not share any members with itself. No set can contain itself.

Natural Numbers ℕ

Von Neumann 1923

succ(a) ≝ a ∪ {a}

ℕ ≝ ({}∈ℕ) ∧ (∀a∈ℕ : succ(a)∈ℕ)

Natural numbers are an effective way to represent discrete quantities that can't be separated into smaller components.

Natural Numbers ℕ:

  • 0ℕ ≝ {}
  • 1ℕ ≝ succ(0) = {0} = {{}}
  • 2ℕ ≝ succ(1) = {0, 1} = {{}, {{}}}
  • 3ℕ ≝ succ(2) = {0, 1, 2} = {{}, {{}}, {{}, {{}}}}
  • 4ℕ ≝ succ(3) = {0, 1, 2, 3} = ...
Most of the time we will use a number without the natural number symbol.

Natural Numbers ℕ: Equality

Every natural number is a distinct set so set equality works for testing natural number equality.

Natural Numbers ℕ: Order

We can compare natural numbers using set membership and equality.

  • ∀a,b∈ℕ a < b ≝ a ∈ b
  • ∀a,b∈ℕ a ⩽ b ≝ (a = b) ∨ (a ∈ b)
  • ∀a,b∈ℕ a > b ≝ b ∈ a
  • ∀a,b∈ℕ a ⩾ b ≝ (a = b) ∨ (b ∈ a)

Addition ℕ

Let's define addition which maps two natural numbers to another natural number. We will represent it with a + sign between two natural numbers: a + b (Grassman 1861):

  • Base: ∀a∈ℕ a + 0 ≝ a
  • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)

Addition ℕ Example: 3 + 2

  • 3 + 2 = 3 + succ(1) :: Definition of 2
  • 3 + 2 = 3 + succ(succ(0)) :: Definition of 1
  • 3 + 2 = succ(3 + succ(0)) :: Step +
  • 3 + 2 = succ(succ(3 + 0)) :: Step +
  • 3 + 2 = succ(succ(3)) :: Base +
  • 3 + 2 = succ(4) :: Definition of 4
  • 3 + 2 = 5 :: Definition of 5
We can use this method to add any two natural numbers: break down the number on the left into successors, use the definition step to bring the successors outside, remove the zero and then evaluate the successors. To do this more efficiently we can memorize or use a lookup table.

Addition ℕ: Closed

∀a,b∈ℕ : a + b ∈ℕ

Addition is constructed entirely from successor operations. Since the successor of any natural number is also a natural number, addition is closed.

Addition ℕ: Identity

∀a∈ℕ a + 0 = a = 0 + a

  • 0 + 0 = 0 :: Base +
  • ∃a∈ℕ : 0 + a = a :: Assumption
  • 0 + succ(a) = succ(0 + a) :: Step +
  • 0 + succ(a) = succ(a) :: Induction

Addition ℕ: Associative

∀a,b,c∈ℕ : (a + b) + c = a + (b + c)

  • (a + b) + 0 = a + b = a + (b + 0) :: Base +
  • ∃c∈ℕ ∀a,b∈ℕ : (a + b) + c = a + (b + c)
  • (a + b) + succ(c) = succ((a + b) + c) :: Step +
  • (a + b) + succ(c) = succ(a + (b + c)) :: Induction
  • (a + b) + succ(c) = a + succ(b + c) :: Step +
  • (a + b) + succ(c) = a + (b + succ(c)) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ : a + b = b + a

We will prove two simpler results and make use of them to prove that addition is commutative

  • Successor + 1: ∀a∈ℕ succ(a) = a + 1
  • Commutative 1: ∀a∈ℕ a + 1 = 1 + a

Addition ℕ: Successor + 1

∀a∈ℕ succ(a) = a + 1

  • succ(a) = succ(a + 0) :: Base +
  • succ(a) = a + succ(0) :: Step +
  • succ(a) = a + 1 :: Definition of 1

Addition ℕ: Commutative 1

∀a∈ℕ a + 1 = 1 + a

  • 0 + 1 = 1 + 0 :: Identity +
  • ∃a∈ℕ : a + 1 = 1 + a :: Assumption
  • succ(a) + 1 = succ(a) + succ(0) :: Definition of 1
  • succ(a) + 1 = succ(succ(a) + 0) :: Step +
  • succ(a) + 1 = succ((a + 1) + 0) :: Successor + 1
  • succ(a) + 1 = succ(a + 1) :: Base +
  • succ(a) + 1 = succ(1 + a) :: Induction
  • succ(a) + 1 = 1 + succ(a) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ : a + b = b + a

  • ∃b∈ℕ ∀a∈ℕ : a + b = b + a :: Assumption
  • a + succ(b) = a + (b + 1) :: Successor + 1
  • a + succ(b) = (a + b) + 1 :: Associativity
  • a + succ(b) = (b + a) + 1 :: Induction
  • a + succ(b) = b + (a + 1) :: Associativity
  • a + succ(b) = b + (1 + a) :: Communtative 1
  • a + succ(b) = (b + 1) + a :: Associativity
  • a + succ(b) = succ(b) + a :: Successor + 1

Addition ℕ

  • Definition:
    • Base: ∀a∈ℕ a + 0 ≝ a
    • Step: ∀a,b∈ℕ a + succ(b) ≝ succ(a + b)
  • Closed: ∀a,b∈ℕ : a + b ∈ℕ
  • Identity: ∀a∈ℕ a + 0 = a = 0 + a
  • Associative: ∀a,b,c∈ℕ : (a + b) + c = a + (b + c)
  • Commutative: ∀a,b∈ℕ : a + b = b + a

Multiplication ℕ

Let's define multiplication which maps two natural numbers to another natural number. We will represent it with a × sign between two natural numbers: a × b (Grassman 1861):

  • Base: ∀a∈ℕ a × 0 ≝ 0
  • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)

Multiplication ℕ Example: 3 × 2

  • 3 × 2 = 3 × succ(1) :: Definition of 2
  • 3 × 2 = 3 × succ(succ(0)) :: Definition of 1
  • 3 × 2 = 3 + (3 × succ(0)) :: Step ×
  • 3 × 2 = 3 + (3 + (3 × 0)) :: Step ×
  • 3 × 2 = 3 + (3 + 0) :: Base ×
  • 3 × 2 = 3 + 3 :: Addition
  • 3 × 2 = 6 :: Addition
We haven't shown that 3 plus 3 is 6 (the successor of 5). That's left as an exercise for the audience.

Multiplication ℕ: Closed

∀a,b∈ℕ : a × b ∈ℕ

Multiplication is constructed entirely from successor operations and addition. Since the successor of any natural number is also a natural number and addition is closed over the natural numbers, multiplication is also closed.

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

Right identity:

  • a × 1 = a × succ(0) :: Definition of 1
  • a × 1 = a + (a × 0) :: Step ×
  • a × 1 = a + 0 :: Base ×
  • a × 1 = a :: Addition ℕ

Multiplication ℕ: Identity

∀a∈ℕ a × 1 = a = 1 × a

Left identity:

  • 1 × 0 = 0 :: Base ×
  • ∃a∈ℕ 1 × a = a :: Assumption
  • 1 × succ(a) = 1 + (1 × a) :: Step ×
  • 1 × succ(a) = 1 + a :: Induction
  • 1 × succ(a) = succ(a) :: Addition ℕ

Multiplication ℕ: Distributive

∀a,b,c∈ℕ : ((a + b) × c = (a × c) + (b × c)) ∧
(c × (a + b) = (c × a) + (c × b))

When we multiply a sum we can push the product through to each term. Because equality is symmetric we can also reverse this by pulling a common product out of each term.

Multiplication ℕ: Distributive

Let's separate this property into right and left properties we can prove separately.

  • Right: ∀a,b,c∈ℕ : ((a + b) × c = (a × c) + (b × c))
  • Left: ∀a,b,c∈ℕ : (c × (a + b) = (c × a) + (c × b))

Multiplication ℕ: Distributive

∀a,b,c∈ℕ : (a + b) × c = (a × c) + (b × c)

Right distributive special case: c = 0

  • (a + b) × 0 = 0 :: Base ×
  • (a + b) × 0 = 0 + 0 :: Addition ℕ
  • (a + b) × 0 = (a × 0) + (b × 0) :: Base ×

Multiplication ℕ: Distribute

  • (a + b) × 0 = (a × 0) + (b × 0)
  • ∃c∈ℕ ∀a,b∈ℕ : ((a + b) × c = (a × c) + (b × c)) :: Assumption
  • ((a + b) × succ(c) = (a + b) + ((a + b) × c) :: Step ×
  • ((a + b) × succ(c) = (a + b) + ((a × c) + (b × c)) :: Induction
  • ((a + b) × succ(c) = (a + (a × c)) + (b + (b × c)) :: Addition ℕ
  • ((a + b) × succ(c) = (a × succ(c)) + (b × succ(c)) :: Step ×

Multiplication ℕ: Distributive

∀a,b,c∈ℕ : c × (a + b) = (c × a) + (c × b)

Left distributive special case: c = 0

  • c × (a + 0) = c × a :: Addition ℕ
  • c × (a + 0) = (c × a) + 0 :: Addition ℕ
  • c × (a + 0) = (c × a) + (c × 0) :: Definition Base

Multiplication ℕ: Distributive

  • c × (a + 0) = (c × a) + (c × 0)
  • ∃c∈ℕ ∀a,b∈ℕ : c × (a + b) = (c × a) + (c × b) :: Assumption
  • c × (a + succ(b)) = c × succ(a + b) :: Addition ℕ
  • c × (a + succ(b)) = c + (c × (a + b)) :: Step ×
  • c × (a + succ(b)) = c + ((c × a) + (c × b)) :: Induction
  • c × (a + succ(b)) = (c × a) + (c + (c × b)) :: Addition ℕ
  • c × (a + succ(b)) = (c × a) + (c × succ(b)) :: Step ×

Multiplication ℕ: Associative

∀a,b,c∈ℕ : (a × b) × c = a × (b × c)

Special case: c = 0

  • (a × b) × 0 = 0 :: Base ×
  • (a × b) × 0 = a × 0 :: Base ×
  • (a × b) × 0 = a × (b × 0) :: Base ×

Multiplication ℕ: Associative

  • (a × b) × 0 = a × (b × 0)
  • ∃c∈ℕ ∀a,b∈ℕ : (a × b) × c = a × (b × c) :: Assumption
  • (a × b) × succ(c) = (a × b) + ((a × b) × c) :: Step ×
  • (a × b) × succ(c) = (a × b) + (a × (b × c)) :: Induction
  • (a × b) × succ(c) = a × (b + (b × c)) :: Distributive ×
  • (a × b) × succ(c) = a × (b × succ(c)) :: Step ×

Multiplication ℕ: Commutative

∀a,b∈ℕ : a × b = b × a

Special case: b = 0

  • 0 × 0 = 0 :: Base ×
  • ∃a∈ℕ 0 × a = 0 :: Assumption
  • 0 × succ(a) = 0 + (0 × a) :: Step ×
  • 0 × succ(a) = 0 + 0 :: Induction
  • 0 × succ(a) = 0 :: Addition ℕ
  • 0 × succ(a) = succ(a) × 0 :: Base ×

Multiplication ℕ: Commutative

  • a × 0 = 0 × a
  • ∃b∈ℕ ∀a∈ℕ : a × b = b × a :: Assumption
  • a × succ(b) = a + (a × b) :: Step ×
  • a × succ(b) = a + (b × a) :: Induction
  • a × succ(b) = (b × a) + a :: Addition ℕ
  • a × succ(b) = (b × a) + (1 × a) :: Identity
  • a × succ(b) = (b + 1) × a :: Distributive
  • a × succ(b) = succ(b) × a :: Successor + 1

Multiplication ℕ

  • Definition:
    • Base: ∀a∈ℕ a × 0 ≝ 0
    • Step: ∀a,b∈ℕ a × succ(b) ≝ a + (a × b)
  • Closed: ∀a,b∈ℕ : a × b ∈ℕ
  • Identity: ∀a∈ℕ a × 1 = a = 1 × a
  • Distributive: ((a + b) × c = (a × c) + (b × c)) ∧
    (c × (a + b) = (c × a) + (c × b))
  • Associative: ∀a,b,c∈ℕ : (a × b) × c = a × (b × c)
  • Commutative: ∀a,b∈ℕ : a × b = b × a

Cardinality

Now that we have sets and natural numbers we can develop a way to talk about how many elements a set contains. We will need a few more tools.

Extensionality means that members of a set have no meaningful order.

∀a,b : {a, b} = {b, a}

Let's impose some order on them!

Ordered Pairs

Kuratowski 1921: ⟨a, b⟩ ≝ {{a}, {a, b}}

  • a is first in ⟨a, b⟩ ≝ ∀c∈⟨a, b⟩ : a∈c
  • b is second in ⟨a, b⟩ ≝ (∃c∈⟨a, b⟩ : b∈c) ∧ (∀c,d∈⟨a, b⟩ : c ≠ d ⟹ b≠c ∧ b≠d)
  • ∀a,b (a ≠ b) ⟺ (⟨a, b⟩ ≠ ⟨b, a⟩)
  • ∀a,b (a ≠ b) ⟺ {{a}, {a, b}} ≠ {{b}, {a, b}}

Relation

A relation maps members of one set to another. Of course a relation is a set too.

(r : D ⟶ R) ≝ { (a, b) : a∈D,b∈R }

  • D is the Domain of the Relation
  • R is the Range of the Relation
  • Application: r(a) = b ⟺ (a, b)∈r

Meet the Jections

Given relation r : D ⟶ R

  • Injection: ∀a,b∈D : r(a) = r(b) ⟹ a = b
    No two members of D relate to the same member of R
  • Surjection: ∀a∈R ∃b∈D : r(b) = a
    Every member of R relates from at least one member of D
  • Bijection: both an injection and a surjection

Cardinality

If a bijection between a natural number and some other set then we say that other set has that many elements. For example, if a bijection exists between set a and the natural number 3 (which is the set {{}, {{}}, {{}, {{}}}}) we say that a has three elements.

Infinite Cardinality

If a bijection between a set b and the set of natural numbers itself exists then we say that b has a countably infinite number of elements.

Integers ℤ

ℤ ≝ { ∀a,b∈ℕ : ⟨a, b⟩ }

∀a,b∈ℕ [a - b] ≝ {∀⟨c1, c2⟩∈ℤ : c1 + b = c2 + a}

Integers extend the natural numbers. They introduce inverses for addition and negative numbers which can represent concepts like debt.

Integers ℤ: Equality

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
(⟨a1, a2⟩ =ℤ ⟨b1, b2⟩) ≝ a1 + b2 = a2 + b1

  • Reflexive: ∀⟨a1, a2⟩∈ℤ : a1 + a2 = a1 + a2
  • Symmetric: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ :
    a1 + b2 = b1 + a2 ⟺ b1 + a2 = a1 + b2
  • Transitive: ∀⟨a1, a2⟩,⟨b1, b2⟩,⟨c1, c2⟩∈ℤ :
    (a1 + c2 = c1 + a2) ∧ (b1 + c2 = c1 + b2) ⟺
    a1 + b2 = a2 + b1

Integers ℤ: Order

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
(⟨a1, a2⟩ < ⟨b1, b2⟩) ≝ a1 + b2 < a2 + b1

Order for integers is defined in terms of the order of the underlying natural numbers.

Addition ℤ

Let's define addition which maps two integers to another integer. As with natural numbers we will represent this with a + sign between two natural numbers: a + b

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
⟨a1, a2⟩ + ⟨b1, b2⟩ ≝ ⟨a1 + b1, a2 + b2⟩

Addition ℤ Example: 3 + 2

  • 3ℤ + 2ℤ = ⟨3, 0⟩ + ⟨5, 3⟩
  • 3ℤ + 2ℤ = ⟨3 + 5, 0 + 3⟩
  • 3ℤ + 2ℤ = ⟨8, 3⟩
  • 3ℤ + 2ℤ = ⟨5, 0⟩
  • 3ℤ + 2ℤ = 5ℤ

    • We can reduce an integer to its simplest form by finding the predecessor of each member of the ordered pair until one or the other is zero.

Addition ℤ Example: 3 + (-2)

  • 3ℤ + (-2ℤ) = ⟨3, 0⟩ + ⟨0, 2⟩
  • 3ℤ + (-2ℤ) = ⟨3 + 0, 0 + 2⟩
  • 3ℤ + (-2ℤ) = ⟨3, 2⟩
  • 3ℤ + (-2ℤ) = ⟨1, 0⟩
  • 3ℤ + (-2ℤ) = 1ℤ
Here's another example, this time with a negative integer. Nothing changes and the result matches our intuition: this takes two away from three.

Addition ℤ: Closed

∀a,b∈ℤ : a + b ∈ℤ

Addition for integers is constructed entirely from natural number addition. The result of adding two integers is an ordered pair consisting of sums of natural numbers.

Addition ℤ: Identity

∀a∈ℤ : a + 0 = a = 0 + a

  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨a1 + 0, a2 + 0⟩ :: Definition
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨a1, a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨0 + a1, 0 + a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨0, 0⟩ + ⟨a1, a2⟩ :: Definition

Addition ℤ: Associative

∀a,b,c∈ℤ : (a + b) + c = a + (b + c)

  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨(a1 + b1) + c1, (a2 + b2) + c2)⟩ :: Definition +
  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨a1 + (b1 + c1), a2 + (b2 + c2)⟩ :: Addition ℕ
  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨a1, a2⟩ + (⟨b1, b2⟩ + ⟨c1, c2)⟩ :: Definition +
As you can see, all of the heavy lifting is done by natural number addition.

Addition ℤ: Commutative

Rational Numbers ℚ

Integers are a useful extension of natural numbers, but while they introduce additive inverses they don't have multiplicative inverses. Rational numbers have inverses for both addition and multiplication with one important exception. The additive identity has no multiplicative inverse. This happens because all rational numbers multiplied by zero collapse into zero. There's no way to reverese this since the information about which number was multiplied by zero has been lost.

A rational number is an ordered pair of natural numbers. Just as there are an infinite number of ordered pairs of natural numbers which represent the same integer, there are an infinite number of ordered pairs of integers which represent the same natural number. We will extend addition to the rational numbers by adding the integers that make up the rational numbers being added. All of the properties of integers addition will be preserved, including inverses.

Multiplication ℚ

We will extend multiplication to the rational numbers by multiplying the integers that make up the rational numbers in question. All of the properties of integers multiplication will be preserved. Because multiplication by zero takes all numbers to zero there's no way to go backwards from zero.

Real Numbers ℝ

Rational numbers can represent most quantities that are important but there are some important exceptions. These exceptions are called irrational numbers because they are not rational.

Irrational numbers include almost all roots. Roots are numbers which can be multiplied by themselves some number of times to produce rational numbers. Some roots are rational. For example, 2 × 2 = 4, which means that 2 is the square root of 4. Similarly, 3 × 3 × 3 = 27, which means 3 is the cube root of 27. However, there is no rational number that can be multiplied by itself to get 2.

Suppose this was not true. If there is a rational number which when multiplied by itself results in the rational numbers 2, which can be represented by a pair of integers such as (2, 1) or (416, 208). This means there is a pair of integers, (a, b), such that (a × a, b × b) is equal to (2, 1).

All integers are either even or odd. If an integer a is even then there is another integer a' which can be multiplied by 2 to produce a. For example, 6 is even and 3 can be multiplied by 2 to produce 6. Given a rational number (a, b) which when multiplied by itself equals rational number 2 then we can ask whether a and b are even or odd. If both a and b are even, then we can replace them with a' and b' such that a' × a' = a and b' × b' = b. (a', b') = (a, b) so (a', b') × (a', b') = (2, 1). We can repeat this until either a or b is odd.

According to the definition of rational number multiplication, integers a × a = 2 × b × b; This means a × a must be even, since b × b multiplied by 2 is equal to it. Any odd number multiplied by an odd number is odd, so a must also be even. That means there is an integer a' which when multiplied by 2 is a. So we can replace a with 2 × a': (2 × a') × (2 × a') = 2 × b × b; Multiplication is commutative so: 2 × 2 × a' × a' = 2 × b × b; We can cancel 2 from both sides so: 2 × a' × a' = b × b. This means b × b is even, which means b is even. So a and b are both even, despite what we did earlier to ensure that at least one is odd. This is a contradiction so there can't be a rational number that satisfies this equation.

No rational number can be the root of 2 but we can get arbitrarily close. Let's start with 1. 1 × 1 = 1, which is smaller than 2. We can multiply 2 by 2, but this is 4 which is larger than 2. We can try (3, 2) which squares to (9, 4) but that's larger than (8, 4) which is equal to 2. So we could try (5, 4) which squares to (25, 16) but that's smaller than (32, 16), which is equal to 2. We can continue like this indefinitely. We'll never reach 2 exactly but we'll get closer and closer. In fact we can separate the rational numbers into all the numbers larger than the square root of 2 and all the numbers smaller. This is called a Dedekind cut.

  • Definition: A real number is a cut (Dedekind 1858-1862) which separates all rational numbers into one of two sets. One set contains all the rational numbers equal to or larger than the number we're looking for. The other contains all rational numbers smaller than it. Every rational number can be represented by a real number, but every irrational number can also be represented this way.
  • There are more real numbers than natural numbers, integers or rational numbers (TODO describe Cantor diagonalization)
  • Unlike other kinds of numbers we've described we will not be able to explicitly manipulate real numbers. We can manipulate the rational parts using addition and multiplication but the irrational parts must be carried around. For example, we could talk about 1 + √2. We can't simplify this any further. However, if we multiply it by 1 - √2 we get 1 + √2 × √2 which is 1 + 2 = 3.

Chapter 2: Abstract Algebra

Magma (M, ∘)

A magma is a set M and a binary operation M ∘ M ⟹ M with one important property:

  • Closed: ∀a,b∈M : a ∘ b ∈M

Groups (G, ∘)

A group is a set G and a binary operation G ∘ G ⟹ G with four properties:

  • Closed: ∀a,b∈G : a ∘ b ∈G
  • Associative: (a ∘ b) ∘ c = a ∘ (b ∘ c)
  • Identity: ∃e∈G : ∀a∈G a ∘ e = a
  • Inverses: ∀a∈G ∃a-1∈G : a ∘ a-1 = e

The definition of a group requires that the identity and inverses work when applied from the right. Without knowing anything else about the group we can prove that these values work from the left as well.

  • e = a-1 ∘ (a-1)-1 (Inverses)
  • e = (a-1 ∘ e) ∘ (a-1)-1 (Identity)
  • e = (a-1 ∘ (a ∘ a-1)) ∘ (a-1)-1 (Inverses)
  • e = ((a-1 ∘ a) ∘ a-1) ∘ (a-1)-1 (Associativity)
  • e = (a-1 ∘ a) ∘ (a-1 ∘ (a-1)-1) (Associativity)
  • e = (a-1 ∘ a) ∘ e (Identity)

So inverses work regardless of which side they're applied. This means the identity also works from both sides:

  • e ∘ a = (a ∘ a-1) ∘ a (Inverses)
  • e ∘ a = a ∘ (a-1 ∘ a) (Associativity)
  • e ∘ a = a ∘ e (Inverses)
  • e ∘ a = a (Identity)

A group is the simplest mathematical structure that allows algebra. Suppose we have know a, b and x ∘ a = b:

  • (x ∘ a) ∘ a-1 = b ∘ a-1 (Transitive Equality)
  • x ∘ (a ∘ a-1) = b ∘ a-1 (Associativity)
  • x ∘ e = b ∘ a-1 (Inverses)
  • x = b ∘ a-1 (Identity)

Abelian Groups

An abelian group is a group with a commutative operation. Integers with addition form an abelian group: (ℤ, +).

Rings (R, ∘, ⊚)

A ring is a set and two binary operations R ∘ R ⟹ R and R ⊚ R ⟹ R with four properties:

  • (R, ∘) is an abelian group
  • Closed: ∀a,b∈R : a ⊚ b ∈ R
  • Associative: (a ⊚ b) ⊚c = a ⊚ (b ⊚ c)
  • Distributive: ∀a,b,c∈R :
    (a ⊚ (b ∘ c) = (a ⊚ b) ∘ (a ⊚ c)) ∧
    ((b ∘ c) ⊚ a = (b ⊚ a) ∘ (c ⊚ a))

Commutative Rings

A commutative ring is a with a commutative operations. Integers with addition and multiplication form a commutative ring (ℤ, +, ×).

Unitary Rings (U, ∘, ⊚)

A unitary ring is a ring with an identity element for the second operation:

  • (U, ∘, ⊚) is a ring
  • Identity: ∃e∈U ∀a∈U : a ⊚ e = a

Integers with addition and multiplication form a unitary ring (ℤ, +, ×).

Fields (F, ∘, ⊚)

A field is a unitary ring with inverses:

  • (F, ∘, ⊚) is a unitary ring
  • Inverses: ∀a∈U a ≠ e∘ ⟹ ∃a-1∈U : a ⊚ a-1 = e⊚

An example of a field is rational numbers with addition and multiplication: (ℚ, +, ×)
