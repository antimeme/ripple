Just in case Tweedledee's explanation isn't enough, let's go over a few things. Logic is a complex topic and addresses many concepts in the wider world. We only need a few relatively simple ideas here.

Informal Logic

A familiar argument:

  • Socrates is a man.

Formal Logic

Formal Systems

  • Theses: things that are formally true
    • Axioms: foundational ideas accepted as true
    • Theorem: proven using axioms and other theorems according to rules of inference
  • Rules of Inference: acceptable ways to combine theses to make new ones
  • Definitions (≝): short hand that can be replaced with more fundamental concepts

Formal Systems: Properties

Desirable properties of a formal system include:

  • Consistent: no statement can be proven both true and false
  • Complete: all statements can be proven either true or false
  • Useful: statements match measurable physical processes and make falsifiable preditions

Incompleteness

Any sufficiently powerful formal system is either inconsistent or incomplete. (Gödel 1931 CE)

Propositional Calculus

Propositional calculus (Frege 1879 CE and Łukasiewicz 1930 CE) deals with statements which are either true or false.

Propositional calculus is sometimes called zero-order logic. First proposed by Frege in 1879, propositional calculus was improved by woo-kah-SHAY-vitch and others starting around 1930.

  • Example: It's raining outside
  • Example: The ground is wet
  • Example: You should carry an umbrella

Real examples are complicated by things like time, geography, scale and opinion. Propositional calculus has no tools for dealing with these.

Statements

Every statement in propositional calculus is can be classified as one of the following:

Theses and Tautologies

All theses of propositional calculus are tautologies. All tautologies are theses of propositional calculus, which is both consistent and complete. This is possible because propositional calculus is not complex enough to provoke Gödel.

Rule of Substitution

Substitution allows us to construct theorems by replacing variables in an established thesis with any well formed statement.

P ⊢ P (Q := Statement, ...)

Substitution: Examples

  • P ⟹ (Q ⟹ P) ⊢ P ⟹ (P ⟹ P) :: Q := P
  • P ⟹ (Q ⟹ P) ⊢ Q ⟹ (P ⟹ Q) :: P := Q, Q := P
  • P ⟹ (Q ⟹ P) ⊢ R ⟹ (S ⟹ R) :: P := R, Q := S
  • P ⟹ (Q ⟹ P) ⊢ (R ∧ S) ⟹ (¬R ⟹ (R ∧ S)) ::
    P := (R ∧ S), Q := ¬R

There's no way to make a tautology false so we can substitute any statement for its variables.

Rule of Replacement

Replacement allows us to construct theorems either by replacing expressions in a thesis with definitions or replacing definitions with their underlying meanings.

Replacement: Examples

  • T ⊼ (T ⊼ T) ⊢ T ⟹ T :: (P ⟹ Q) ≝ P ⊼ (Q ⊼ Q)
  • P ⟹ (Q ⟹ P) ⊢ P ⟹ (Q ⊼ ¬P) :: Implication
  • P ⟹ (Q ⟹ P) ⊢ P ⊼ ¬(Q ⊼ ¬P) :: Implication
  • P ⟹ (Q ⟹ P) ⊢ P ⊼ ((Q ⊼ (P ⊼ P)) ⊼ (Q ⊼ (P ⊼ P))) :: Negation

Replacing a definition with its value or a value with a definition leaves it unchanged.

Rule of Detatchment

Detatchment allows us to construct theorems by extracting part of a thesis once we've established another.

P ⊼ (Q ⊼ R), P ⊢ R

We can't apply the rule of detatchment if P is false so that part of the truth table is omitted. The remaining rows show that Q and R must be true.

Modus Ponens

P ⟹ Q, P ⊢ Q

Modus ponens follows from our detatchment rule.

  • P ⊼ (Q ⊼ R), P ⊢ R :: Detatchment
  • P ⊼ (Q ⊼ Q), P ⊢ Q :: Substitution (R := Q)
  • P ⊼ ¬Q, P ⊢ Q :: Replacement (Negation)
  • P ⟹ Q, P ⊢ Q :: Replacement (Implication)
This is a more familiar form of modus ponens. It follows directly from Nicod's modus ponens. When Q and R are the same the two Sheffer strokes convert to implication. This is sometimes called implication elmination because it converts an implication to its consequent, using the known truth of the antecedant.

Predicate Calculus

A predicate is a proposition that is true or false based on the value of its variables. This is also called first order logic.

What is a Predicate?

  • Example: A _____ is a fruit
  • An apple is a fruit (true)
  • A carrot is a fruit (false)

An apple isn't true or false. It's an object rather than a statement. This distinction is important.

A predicate is a statement with one or more holes in it. Depending on what we put in the holes it may be true or false. If our predicate is blank is a fruit then it's true if we fill in the blank with apple but false if we use carrot instead.

Cast of Characters

  • Object: ♠, ♥, ♣, ♦
  • Variable (Object): a, b, c, ...
  • Function (Object): f(a), g(a, b), h, ...
    • next(♠) ⟶ ♥
    • next(♥) ⟶ ♣
    • kestral(♠, ♣) ⟶ ♠
    • kestral(♦, ♣) ⟶ ♦
  • Predicate (true or false): P, Q(a), R(a, b), ...
    • BLACK(♠) : true
    • BLACK(♥) : false
    • SAME(♠, ♠) : true
    • SAME(♠, ♣) : false

Universal Quantifier

∀a P(a)

A universal quantifier asserts that a predicate is true for all possible values of a variable. This binds a variable so that it is no longer free in the predicate.

Existential Quantifier

∃a P(a) ≝ ¬∀a ¬P(a)

From the universal quantifier we construct the existential quantifier. This asserts that some value of a variable satisfies the predicate.

The negation of this quantifier is sometimes useful.

∄a P(a) ≝ ¬∃a P(a) ⟺ ∀a ¬P(a)

Predicate Calculus

Rule of Generalization: P ⊢ ∀a P
Quantified Implication:
(∀a (P ⟹ Q)) ⟹ ((∀a P) ⟹ (∀a Q))
Distinctness: P ⟹ ∀a P
Existance: ¬∀a ¬(a = b)
Equality: (a = b) ⟹ ((a = c) ⟹ (b = c))

Liebniz's Law

∀a,b (a = b) ⟹ (P(a) ⟺ P(b))

When a and b are equal the value of any predicate using a is the same as the same predicate using b. We can substitute equal values without changing any results.

Nothing But Set

We will use set theory to construct relations, functions, numbers, arithmetic and algebra. Set theory is a formal system in which the only objects are sets and the only thing sets can do is contain other objects. (Zermelo-Fraenkel 1908-1922 CE)

Axiomatic set theory is the foundation of this course. So far we've covered the most important rules of inference. This seciton will explain the axioms of our formal system. The axioms described here were first developed by Ernst Zermelo, Abraham Fraenkel and Thorlaf Skolem. Alternative formulations, such as Von Neumann's Category Theory, also exist but we won't use them here.

Axiom of Extensionality

∀a,b (a = b) ⟺ (∀c c∈a ⟺ c∈b)

Two sets that contain the same elements are the same set.

{ ♣, ♣, ♦, ♠, ♥ } = { ♠, ♥, ♣, ♦ }

Extensionality means that any two sets that contain the same elements are the same set.

Axiom (Schema) of Specification

∀a,P ∃b ∀c c∈b ⟺ (c∈a ∧ P(c))

Given a set and a predicate there is a set which contains only members for which the predicate is true. Any predicate can filter any set.

Notation: { a∈b : P(a) }

Specification is actually an axiom schema because there is technically one axiom for every possible predicate. This schema allows us to use any predicate to filter the elmeents of any set.

Empty Set

{} ≝ ∀a a∉{}

The empty set doesn't contain anything.

Extensionality means this set must be unique.

If any set exists specification allows us to prove that the empty set also exists:

∃a ⟹ ∃{ b∈a : b ≠ b } = {}

Axiom of Pairing

∀a,b ∃{a, b} ∀c (c∈{a, b} ⟺ (c=a ∨ c=b))

Any two members or sets can be used to create a set which contains only those two sets: { ♠, ♣ }

Elements of the pair may be equal. Extensionality means that there only a single member:

{ ♠, ♠ } = { ♠ }

Pairing allows us to use any two sets to create a third set with only those two sets as members. We can use the same set twice as members to create a set with only one member.

Axiom of Union

∀a ∃∪a ∀b∈∪a (∃c∈a b∈c)

Any set can be used to create a set by combining all the members of its members.

∪{ {♠, ♥}, {♥, ♦, ♣} } = {♠, ♥, ♣, ♦}

∀a,b (a ∪ b) ≝ ∪{ a, b }

Union allows us to combine the members of the members of a set. We can use this along with the axiom of pairing to create a union operator which combines two sets into one.

Intersection

∀a ∩a ≝ { c∈∪a : ∀b∈a c∈b }

The intersection of a set of sets contains all members that are present in all the members. This can be defined on a pair of sets.

∀a,b (a ∩ b) ≝ ∩{ a, b }

{ ♠, ♥ } ∩ { ♠, ♦ } = { ♠ }

Difference

∀a,b (a ⊖ b) ≝ { c∈a : c∉b }

The difference between two sets contains only the elements of the first that are not in the second.

{ ♠, ♥, ♣ } ⊖ { ♥, ♦ } = { ♠, ♣ }

Pairing and union axioms allow us to create sets of any size from whatever elements exist.

  • ∪{ { ♠, ♥ }, { ♦, ♣ } } = { ♠, ♥, ♣, ♦ }

Union Properties

These easily follow from extensionality.

  • Identity: ∀a (a ∪ {}) = ∪{ a, {} } = a
  • Idempotent: ∀a (a ∪ a) = ∪{ a, a } = a
  • Communtative: ∀a,b (a ∪ b = b ∪ a) ⟺
    (∪{a, b} = ∪{b, a})
  • Associative: ∀a,b,c ((a ∪ b) ∪ c = a ∪ (b ∪ c)) ⟺
    (∪{ ∪{ a, b }, c } = ∪{ a, ∪{ b, c } })

Subsets

∀a,b (a ⊆ b) ≝ (∪{a, b} = b)

One set is a subset of another if the union operator leaves the other set unchanged.

∀a,b (a ⊆ b) ⟹ ∀c (c∈a ⟹ c∈b)

We'll use a sideways letter U with a line under it to indicate that one set is a subset of a second set, which means all members of the first set are also members of the second. Notice that every set is a subset of itself.

Subset Properties

These easily follow from extensionality.

  • Identity: ∀a {} ⊆ a
  • Reflexive: ∀a a ⊆ a
  • Antisymmetric: ∀a,b (a ⊆ b) ∧ (b ⊆ a) ⟹ (a = b)
  • Transitive: ∀a,b,c (a ⊆ b) ∧ (b ⊆ c) ⟹ (a ⊆ c)

Axiom of Power Set

∀a ∃℘a ∀b (b∈℘a ⟺ b⊆a)

The subsets of any set make up a set of their own.

℘{♠, ♥, ♣, ♦} = {{}, {♠}, {♥}, {♣}, {♦},
{♠, ♥}, {♠, ♣}, {♠, ♦}, {♥, ♣}, {♥, ♦}, {♣, ♦},
{♥, ♣, ♦}, {♠, ♣, ♦}, {♠, ♥, ♣}, {♠, ♥, ♦},
{♠, ♥, ♣, ♦}}

Every set has a power set, which is the set of all subsets of the original set. Adding an element to the original set doubles the size of the power set. Each member of the previous power set must still be included and we must add another member for each of those that contains the new element.

Axiom (Schema) of Replacement

∀a,f ∃b ∀c c∈b ⟺ ∃d∈a f(c) = d

Given a function and a subset of the domain there is a set which contains all the function outputs for all members of the set.

Notation: { f(a) : a∈b }

Replacement is another schema but this time it allows us to use a function to create one set from another. There is one axiom for each possible mapping. We haven't yet explained what a function is or how to construct one. We will do that once we've established a few important tools.

Axiom of Regularity

∀a ∃b∈a ⟹ ∃c∈a ∄d (d∈a ∧ d∈c)

A set that contains any members must contain at least one member that does not share any members with itself. No set can contain itself.

Regularity prevents sets from containing themselves. The form given here prevents chains of sets that eventually come back to themselves and other such tricks. This axiom prevents paradoxes that afflicted earlier set theories.

Axiom of Infinity

∃a (({}∈a) ∧ (∀b∈a ∪{b, {b, b}} ∈ a))

A set with infinitely many distinct members exists. Specifically the set containing the empty set and all sets we can get by taking the union of any member with a set containing only that member exists.

The axiom of infinity guarantees that at least one infinite set exists. This will allow us to create and work with other infinite sets.

Axiom of Choice

∀a {}∉a ⟹ ∃f: a ⟶ ∪a ∀b∈a f(b) ∈ b

For any set that doesn't have any empty members a function exists that chooses one member from each of the sets members.

The axiom of choice has been controversial and it allows some unintuitive results such as the Banach-Tarski theorem. While not necessary for fininte collections of sets, this is useful for working with infinite sets. The famous Banach-Tarski paradox is a consequence of this axiom.

Set Contents

We describe sets using one of these forms.

{} Empty Set: ∀a a∉{}
{ a, b, c } Enumeration
{ a∈b : P(a) } Specification (predicate filter)
{ f(a) : a∈b } Replacement (function outputs)
We usually describe sets in one of these two ways. When a set contains only a few selected members, we enumerate them, separated by commas. Larger sets are difficult to represent this way so we describe them using a predicate which selects members from some other set.

Can you relate?

A relation is a set that embodies a relationship between other sets. Examples of relations include equivalence, order and functions. Relations can also represent complex data.

But First: A Point of Order

Extensionality means that members of a set have no meaningful order.

∀a,b {a, b} = {b, a}

Let's impose some order on them!

Ordered Pairs

⟨a, b⟩ ≝ {{a}, {a, b}} (Kuratowski 1921 CE)

  • ∀a,b (a ≠ b) ⟺ (⟨a, b⟩ ≠ ⟨b, a⟩)

Power Sets and Pairs

All possible ordered pairs of members of a set are themselves members of the power set of the power set of that set.

Given any set and two members of that set, it's clear from the definition of a subset that sets containing one or both of these members are subsets of the original set. Not all members of a power set are ordered pairs, but all ordered pairs are members of the power set.

Equivalence

An equivalence relation has three properties:

  • Reflexive: ∀a ∈ ∪∪≑ (a ≑ a)
  • Symmetric: ∀a,b ∈ ∪∪≑ (a ≑ b) ⟺ (b ≑ a)
  • Transitive: ∀a,b,c ∈ ∪∪≑
    (a ≑ b ∧ b ≑ c) ⟹ (a ≑ c)

This partitions a set into classes that don't overlap.

An equivalence relation has three special properties. Reflexive means every member of the domain relates to itself. Symmetric means we can always reverse the members. Transitive means that whenever there is a chain we can take a shortcut.

Partial Order

A partial order has the following properties:

  • Reflexive: ∀a ∈ ∪∪⋖ (a ⋖ a)
  • Antisymmetric: ∀a,b ∈ ∪∪⋖
    (a ⋖ b) ∧ (b ⋖ a) ⟹ (a ≑ b)
  • Transitive: ∀a,b,c ∈ ∪∪⋖
    (a ⋖ b) ∧ (b ⋖ c) ⟹ (a ⋖ c)
A partial order is a relation on a set that that makes some but not necessarily all pairs of elements comparable. To be a partial order, the relation must be reflexive, antisymmetric and transitive.

Total Order

A total order is a partial order with connex:

  • Reflexive, Antisymmetric and Transitive
  • Connex: ∀a,b ∈ ∪∪⋖ (a ⋖ b) ∨ (b ⋖ a)

This organizes a set into concentric rings.

A total order is a partial order with the connex property, which means that every pair of elements is related in one direction or the other.

Functions

A function is a special kind of relation which maps every member of its domain to the exactly one member of its range.

A function is a relation with the special property that every member of the domain appears in exactly one ordered pair. In other words, functions always map every element in their domains.

Functions

(f : D ⟶ R) ≝ ∀a∈D (∃b∈R ⟨a, b⟩∈f) ∧
(∀b,c∈R (⟨a, b⟩∈f ∧ ⟨a, c⟩∈f) ⟹ b = c)

  • Entire domain mapped: ∀a∈D ∃b∈R ⟨a, b⟩∈f
  • Unique mapping: ∀a∈D ∀b,c∈R
    (⟨a, b⟩∈f ∧ ⟨a, c⟩∈f) ⟹ b = c
  • f(a) = b ⟺ ⟨a, b⟩∈f
We denote function application different than relations. Rather than saying a relates b we say f of a equals b.

Meet the Jections

Given function f : D ⟶ R

  • Injection (one-to-one): Each member of the domain maps to a distinct member of the range
    ∀a,b∈D (f(a) = f(b)) ⟹ (a = b)
  • Surjection (onto): Each member of the range is mapped from at least one member of domain
    ∀b∈R ∃a∈D f(a) = b

Back It Up

Given function f : D ⟶ R

  • Bijection: both an injection and a surjection
  • Inverse: f-1 : R ⟶ D
    ∀a∈D,b∈R (f(a) = b) ⟺ (f-1(b) = a)

Cardinality

Cardinality is a property of every set. We will define it using bijections. The domain and range of any bijection have the same cardinality. If a bijection cannot exist between a pair of sets they have different cardinality.

Cardinality formalizes our intuitive notion of size. For now we will define it indirectly: any pair of sets for which some bijection exists have the same cardinality. We can prove that a pair of sets have different cardinality if we show that no bijection can exist between them.

This is the same as having a domain consisting of ordered pairs or tuples.

Group ⟬G, ∘⟭

A group is a set and a binary function

  • Associative: ∀a,b,c∈G (a ∘ b) ∘ c = a ∘ (b ∘ c)

  • Inverses: ∀a∈G ∃a-1∈G a ∘ a-1 = e∘

A group is the simplest mathematical structure that makes algebra possible.

Abstraction

By definition identity and inverses work when applied from the right.

  • Identity: ∃e∘∈G ∀a∈G a ∘ e∘ = a
  • Inverses: ∀a∈G ∃a-1∈G a ∘ a-1 = e∘

Do these work from the left?

  • ∃e∘∈G ∀a∈G e∘ ∘ a ≟ a
  • ∀a∈G ∃a-1∈G a-1 ∘ a ≟ e∘

Proof: Left Inverses

  • e∘ = a-1 ∘ (a-1)-1 :: Inverses
  • e∘ = (a-1 ∘ e∘) ∘ (a-1)-1 :: Identity
  • e∘ = (a-1 ∘ (a ∘ a-1)) ∘ (a-1)-1 :: Inverses
  • e∘ = ((a-1 ∘ a) ∘ a-1) ∘ (a-1)-1 :: Associativity
  • e∘ = (a-1 ∘ a) ∘ (a-1 ∘ (a-1)-1) :: Associativity
  • e∘ = (a-1 ∘ a) ∘ e∘ :: Inverses
  • e∘ = a-1 ∘ a :: Identity

Proof: Left Identity

  • e∘ ∘ a = (a ∘ a-1) ∘ a :: Inverses
  • e∘ ∘ a = a ∘ (a-1 ∘ a) :: Associativity
  • e∘ ∘ a = a ∘ e∘ :: Left Inverses
  • e∘ ∘ a = a :: Identity

Equation Solving

Suppose we have know a, b and (a ∘ x = b):

  • a-1 ∘ (a ∘ x) = a-1 ∘ b :: Equality
  • (a-1 ∘ a) ∘ x = a-1 ∘ b :: Associativity
  • e∘ ∘ x = a-1 ∘ b :: Inverses
  • x = a-1 ∘ b :: Identity

Order ℕ: Partial Order

Subset relationships are always reflexive, antisymmetric and transitive so this relationship necessarily forms a partial order for natural numbers. We only need the connex property to have a total order.

Induction

Any set that contains 0ℕ and is closed under the successor operation contains all natural numbers. We prove this by showing that something is true for a base case (usually 0ℕ), then assume it's true for some natural number and show that this implies that it's true for its successor.

Successor Subsets

∀a∈ℕ a ⊆ SUCCESSOR(a)

  • a ⊆ a ∪ {a} :: Definition SUCCESSOR
  • a ∪ (a ∪ {a}) = a ∪ {a} :: Definition ⊆

    • All natural numbers are a subset of their successors because any set is a subset of the union of itself with any other set.

Order ℕ: Connex

∀a,b∈ℕ (a ⊆ b) ∨ (b ⊆ a)

  • ∀a ({} ⊆ a) :: Subset Identity
  • ∀a∈ℕ (0ℕ ⊆ a) :: Definition 0ℕ
  • ∀b∈ℕ (0ℕ ⊆ b) ∨ (b ⊆ 0ℕ) :: Base
  • ∃a∈ℕ ∀b∈ℕ (a ⊆ b) ∨ (b ⊆ a) :: Assumption
Remember that the empty set is a subset of any set. Natural number zero is the empty set by definition so it has the connex property when paired with any other natural number. This is our base case for an inductive proof.

Order ℕ: Connex

∀a,b∈ℕ (a ⊆ b) ∨ (b ⊆ a)

  • (b ⊆ a) ⟹ (b ⊆ SUCCESSOR(a)) :: Transitive ⊆
  • (a ⊆ b) ⟹ (a = b) ∨ (a ⊂ b)
  • (a = b) ⟹ (b ⊆ SUCCESSOR(a)) ::
    Successor Subsets
  • (a ⊂ b) ⟹ (SUCCESSOR(a) ⊆ b) :: Definition ℕ
  • (a ⊆ b) ∨ (b ⊆ a) ⟹ (SUCCESSOR(a) ⊆ b) ∨
    (b ⊆ SUCCESSOR(a)) :: Induction

Addition ℕ

Let's define addition which maps two natural numbers to another natural number (Grassman 1861 CE, Peano 1889 CE):

+: ℕ⊗ℕ ⟶ ℕ

  • Base: ∀a∈ℕ a + 0ℕ ≝ a
  • Step: ∀a,b∈ℕ a + SUCCESSOR(b) ≝
    SUCCESSOR(a + b)

Addition ℕ Example: 3 + 2

  • 3ℕ + 2ℕ = 3ℕ + SUCCESSOR(1ℕ) :: Definition of 2ℕ
  • 3ℕ + 2ℕ = 3ℕ + SUCCESSOR(SUCCESSOR(0ℕ)) :: Definition of 1ℕ
  • 3ℕ + 2ℕ = SUCCESSOR(3ℕ + SUCCESSOR(0ℕ)) :: Step +
  • 3ℕ + 2ℕ = SUCCESSOR(SUCCESSOR(3ℕ + 0ℕ)) :: Step +
  • 3ℕ + 2ℕ = SUCCESSOR(SUCCESSOR(3ℕ)) :: Base +
  • 3ℕ + 2ℕ = SUCCESSOR(4ℕ) :: Definition of 4ℕ
  • 3ℕ + 2ℕ = 5ℕ :: Definition of 5ℕ
We can use this method to add any two natural numbers: break down the number on the left into successors, use the definition step to bring the successors outside, remove the zero and then evaluate the successors. To do this more efficiently we can memorize or use a lookup table.

Addition ℕ: Closed

∀a,b∈ℕ a + b ∈ℕ

  • Base: ∀a∈ℕ a + 0ℕ ≝ a
  • Step: ∀a,b∈ℕ a + SUCCESSOR(b) ≝
    SUCCESSOR(a + b)

Addition is constructed entirely from successor operations. Since the successor of any natural number is also a natural number, addition is closed.

Addition ℕ: Identity

∀a∈ℕ a + 0ℕ = a = 0ℕ + a

  • 0ℕ + 0ℕ = 0ℕ :: Base +
  • ∃a∈ℕ 0ℕ + a = a :: Assumption
  • 0ℕ + SUCCESSOR(a) = SUCCESSOR(0ℕ + a) :: Step +
  • 0ℕ + SUCCESSOR(a) = SUCCESSOR(a) :: Induction

Addition ℕ: Associative

∀a,b,c∈ℕ (a + b) + c = a + (b + c)

  • (a + b) + 0ℕ = a + b = a + (b + 0ℕ) :: Base +
  • ∃c∈ℕ ∀a,b∈ℕ : (a + b) + c = a + (b + c)
  • (a + b) + SUCCESSOR(c) = SUCCESSOR((a + b) + c) :: Step +
  • (a + b) + SUCCESSOR(c) = SUCCESSOR(a + (b + c)) :: Induction
  • (a + b) + SUCCESSOR(c) = a + SUCCESSOR(b + c) :: Step +
  • (a + b) + SUCCESSOR(c) = a + (b + SUCCESSOR(c)) :: Step +

Addition ℕ: Commutative

∀a,b∈ℕ a + b = b + a

We will prove two simpler results and make use of them to prove that addition is commutative

  • Successor + 1: ∀a∈ℕ SUCCESSOR(a) = a + 1ℕ
  • Commutative 1: ∀a∈ℕ a + 1ℕ = 1ℕ + a

Addition ℕ: Successor + 1

∀a∈ℕ SUCCESSOR(a) = a + 1ℕ

  • SUCCESSOR(a) = SUCCESSOR(a + 0ℕ) :: Base +
  • SUCCESSOR(a) = a + SUCCESSOR(0ℕ) :: Step +
  • SUCCESSOR(a) = a + 1ℕ :: Definition of 1ℕ

∀a∈ℕ a + 1ℕ = 1ℕ + a

  • ∃a∈ℕ : a + 1ℕ = 1ℕ + a :: Assumption
  • SUCCESSOR(a) + 1ℕ = SUCCESSOR(a) + SUCCESSOR(0ℕ) :: Definition of 1ℕ
  • SUCCESSOR(a) + 1ℕ = SUCCESSOR(SUCCESSOR(a) + 0ℕ) :: Step +
  • SUCCESSOR(a) + 1ℕ = SUCCESSOR((a + 1ℕ) + 0ℕ) :: Successor + 1
  • SUCCESSOR(a) + 1ℕ = SUCCESSOR(a + 1ℕ) :: Base +
  • SUCCESSOR(a) + 1ℕ = SUCCESSOR(1ℕ + a) :: Induction
  • SUCCESSOR(a) + 1ℕ = 1ℕ + SUCCESSOR(a) :: Step +

  • a + SUCCESSOR(b) = a + (b + 1ℕ) :: Successor + 1

  • a + SUCCESSOR(b) = (b + a) + 1ℕ :: Induction
  • a + SUCCESSOR(b) = b + (a + 1ℕ) :: Associativity

  • a + SUCCESSOR(b) = SUCCESSOR(b) + a :: Successor + 1

Multiplication ℕ

Let's define multiplication which maps two natural numbers to another natural number (Grassman 1861 CE, Peano 1889 CE):

×: ℕ⊗ℕ ⟶ ℕ

  • Base: ∀a∈ℕ a × 0ℕ ≝ 0ℕ
  • Step: ∀a,b∈ℕ a × SUCCESSOR(b) ≝ a + (a × b)

Multiplication ℕ Example: 3 × 2

  • 3ℕ × 2ℕ = 3ℕ × SUCCESSOR(1ℕ) :: Definition of 2ℕ
  • 3ℕ × 2ℕ = 3ℕ × SUCCESSOR(SUCCESSOR(0ℕ)) :: Definition of 1ℕ
  • 3ℕ × 2ℕ = 3ℕ + (3ℕ × SUCCESSOR(0ℕ)) :: Step ×
  • 3ℕ × 2ℕ = 3ℕ + (3ℕ + (3ℕ × 0ℕ)) :: Step ×
  • 3ℕ × 2ℕ = 3ℕ + (3ℕ + 0ℕ) :: Base ×
  • 3ℕ × 2ℕ = 3ℕ + 3ℕ :: Addition
  • 3ℕ × 2ℕ = 6ℕ :: Addition
We haven't shown that 3 plus 3 is 6 (the successor of 5). That's left as an exercise for the audience.

Multiplication ℕ: Closed

∀a,b∈ℕ a × b ∈ℕ

  • Base: ∀a∈ℕ a × 0ℕ ≝ 0ℕ
  • Step: ∀a,b∈ℕ a × SUCCESSOR(b) ≝ a + (a × b)

Multiplication is constructed entirely from successor operations and addition. Since the successor of any natural number is also a natural number and addition is closed over the natural numbers, multiplication is also closed.

Multiplication ℕ: Identity

∀a∈ℕ a × 1ℕ = a = 1ℕ × a

Right identity:

  • a × 1ℕ = a × SUCCESSOR(0ℕ) :: Definition of 1ℕ
  • a × 1ℕ = a + (a × 0ℕ) :: Step ×
  • a × 1ℕ = a + 0ℕ :: Base ×
  • a × 1ℕ = a :: Addition ℕ

Multiplication ℕ: Identity

∀a∈ℕ a × 1ℕ = a = 1ℕ × a

Left identity:

  • 1ℕ × 0ℕ = 0ℕ :: Base ×
  • ∃a∈ℕ 1ℕ × a = a :: Assumption
  • 1ℕ × SUCCESSOR(a) = 1ℕ + (1ℕ × a) :: Step ×
  • 1ℕ × SUCCESSOR(a) = 1ℕ + a :: Induction
  • 1ℕ × SUCCESSOR(a) = SUCCESSOR(a) :: Addition ℕ

Multiplication ℕ: Distributive

∀a,b,c∈ℕ ((a + b) × c = (a × c) + (b × c)) ∧
(c × (a + b) = (c × a) + (c × b))

When we multiply a sum we can push the product through to each term. We separate this property into right and left distributive rules.

  • Right: ∀a,b,c∈ℕ ((a + b) × c = (a × c) + (b × c))
  • Left: ∀a,b,c∈ℕ (c × (a + b) = (c × a) + (c × b))

Multiplication ℕ: Distributive

∀a,b,c∈ℕ (a + b) × c = (a × c) + (b × c)

Consider the special case where c = 0ℕ:

  • (a + b) × 0ℕ = 0ℕ :: Base ×
  • (a + b) × 0ℕ = 0ℕ + 0ℕ :: Addition ℕ
  • (a + b) × 0ℕ = (a × 0ℕ) + (b × 0ℕ) :: Base ×

Multiplication ℕ: Distribute

  • (a + b) × 0ℕ = (a × 0ℕ) + (b × 0ℕ)
  • ((a + b) × c = (a × c) + (b × c)) :: Assumption
  • ((a + b) × SUCCESSOR(c) = (a + b) + ((a + b) × c) :: Step ×
  • ((a + b) × SUCCESSOR(c) = (a + b) + ((a × c) + (b × c)) :: Induction
  • ((a + b) × SUCCESSOR(c) = (a + (a × c)) + (b + (b × c)) :: Communtative and Associative ℕ+
  • ((a + b) × SUCCESSOR(c) = (a × SUCCESSOR(c)) + (b × SUCCESSOR(c)) :: Step ×

Multiplication ℕ: Distributive

∀a,b,c∈ℕ c × (a + b) = (c × a) + (c × b)

Consider the special case where b = 0ℕ:

  • c × (a + 0ℕ) = c × a :: Addition ℕ
  • c × (a + 0ℕ) = (c × a) + 0ℕ :: Addition ℕ
  • c × (a + 0ℕ) = (c × a) + (c × 0ℕ) :: Base ×

Multiplication ℕ: Distributive

  • c × (a + SUCCESSOR(b)) = (c × a) + (c × SUCCESSOR(b)) :: Step ×

Multiplication ℕ: Associative

∀a,b,c∈ℕ (a × b) × c = a × (b × c)

Consider the special case where c = 0ℕ:

  • (a × b) × 0ℕ = 0ℕ :: Base ×
  • (a × b) × 0ℕ = a × 0ℕ :: Base ×
  • (a × b) × 0ℕ = a × (b × 0ℕ) :: Base ×

Multiplication ℕ: Associative

  • (a × b) × 0ℕ = a × (b × 0ℕ)
  • ∃c∈ℕ ∀a,b∈ℕ : (a × b) × c = a × (b × c) :: Assumption
  • (a × b) × SUCCESSOR(c) = (a × b) + ((a × b) × c) :: Step ×
  • (a × b) × SUCCESSOR(c) = (a × b) + (a × (b × c)) :: Induction
  • (a × b) × SUCCESSOR(c) = a × (b + (b × c)) :: Distributive ×

∀a,b∈ℕ a × b = b × a

  • 0ℕ × SUCCESSOR(a) = 0ℕ + 0ℕ = 0ℕ :: Base ×

  • ∃b∈ℕ ∀a∈ℕ : a × b = b × a :: Assumption

Integers ℤ

ℤ ≝ { ⟨a, b⟩∈ℕ⊗ℕ }

[a - b] ≝ { ⟨c1, c2⟩∈ℕ⊗ℕ : (a + c2 = c1 + b) }

Integers extend the natural numbers. They introduce inverses for addition and negative numbers which can represent concepts like debt.

Integers ℤ: Equality

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ (⟨a1, a2⟩ =ℤ ⟨b1, b2⟩) ≝
(a1 + b2 = a2 + b1)

  • Reflexive: ∀⟨a1, a2⟩∈ℤ a1 + a2 = a2 + a1
  • Symmetric: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
    a1 + b2 = b1 + a2 ⟺ b1 + a2 = a1 + b2
  • Transitive: ∀⟨a1, a2⟩,⟨b1, b2⟩,⟨c1, c2⟩∈ℤ
    (a1 + b2 = a2 + b1) ∧ (b1 + c2 = b2 + c1) ⟺
    (a1 + c2 = a2 + c1)
Integer equality means that the gap between the numbers has the same number steps in the same direction.

Successor Equality

∀a,b∈ℕ ⟨a, b⟩ =ℤ ⟨SUCCESSOR(a), SUCCESSOR(b)⟩

  • a + SUCCESSOR(b) = SUCCESSOR(a) + b :: Integer Equality
  • a + (b + 1ℕ) = (a + 1ℕ) + b :: Successor + 1
  • (a + b) + 1ℕ = (a + b) + 1ℕ :: Arithmetic ℕ

Every integer has a simplest form, which has at least one member that is natural number zero.

Integers ℤ: Order

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ (⟨a1, a2⟩ ≤ℤ ⟨b1, b2⟩) ≝
(a1 + b2 ≤ℕ a2 + b1)

  • Reflexive: (⟨a1, a2⟩ ≤ℤ ⟨a1, a2⟩) ⟹ (a1 + a2 ≤ℕ a2 + a1)
  • Antisymmetric: (⟨a1, a2⟩ ≤ℤ ⟨b1, b2⟩) ∧
    (⟨b1, b2⟩ ≤ℤ ⟨a1, a2⟩) ⟹ (⟨a1, a2⟩ =ℤ ⟨b1, b2⟩)
  • Transitive: (⟨a1, a2⟩ ≤ℤ ⟨b1, b2⟩) ∧
    (⟨b1, b2⟩ ≤ℤ ⟨c1, c2⟩) ⟹ (⟨a1, a2⟩ ≤ℤ ⟨c1, c2⟩)
  • Connex: (⟨a1, a2⟩ ≤ℤ ⟨b1, b2⟩) ∨ (⟨b1, b2⟩ ≤ℤ ⟨a1, a2⟩)

Addition ℤ

Let's define addition which maps two integers to another integer.

+: ℤ⊗ℤ ⟶ ℤ

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
⟨a1, a2⟩ + ⟨b1, b2⟩ ≝ ⟨a1 + b1, a2 + b2⟩

Addition ℤ Example: 3 + 2

  • 3ℤ + 2ℤ = ⟨3, 0⟩ + ⟨5, 3⟩
  • 3ℤ + 2ℤ = ⟨3 + 5, 0 + 3⟩
  • 3ℤ + 2ℤ = ⟨8, 3⟩
  • 3ℤ + 2ℤ = ⟨5, 0⟩ (Because 8 + 0 = 5 + 3)
  • 3ℤ + 2ℤ = 5ℤ

Addition ℤ Example: 3 + (-2)

  • 3ℤ + (-2ℤ) = ⟨3, 0⟩ + ⟨0, 2⟩
  • 3ℤ + (-2ℤ) = ⟨3 + 0, 0 + 2⟩
  • 3ℤ + (-2ℤ) = ⟨3, 2⟩
  • 3ℤ + (-2ℤ) = ⟨1, 0⟩ (3 + 0 = 1 + 2)
  • 3ℤ + (-2ℤ) = 1ℤ
Here's another example, this time with a negative integer. Nothing changes and the result matches our intuition: this takes two away from three.

Addition ℤ: Isomorphic

∀ a,b∈ℕ ⟨a, 0⟩ + ⟨b, 0⟩ = ⟨a + b, 0 + 0⟩ = ⟨a + b, 0⟩

Adding any two integers that match natural numbers results in the natural number their sum would match.

Addition ℤ: Equivalent

∀ a,b,c∈ℤ (a = b) ⟹ ((a + c) = (b + c))

  • (a = b) ⟹ (a1 + b2 = a2 + b1) :: Equality ℤ
  • a1 + b2 + c1 + c2 = a2 + b1 + c1 + c2 :: Arithmetic ℕ
  • a1 + c1 + b2 + c2 = a2 + c2 + b1 + c1 :: Arithmetic ℕ
  • ⟨a1 + c1, a2 + c2⟩ = ⟨b1 + c1, b2 + c2⟩ :: Definition ℤ
  • ⟨a1, a2⟩ + ⟨c1, c2⟩ = ⟨b1, b2⟩ + ⟨c1, c2⟩ :: Addition ℤ
  • (a = b) ⟹ ((a + c) = (b + c))

Addition ℤ: Closed

∀a,b∈ℤ a + b ∈ℤ

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
⟨a1, a2⟩ + ⟨b1, b2⟩ ≝ ⟨a1 + b1, a2 + b2⟩

Addition for integers is constructed entirely from natural number addition. The result of adding two integers is an ordered pair consisting of sums of natural numbers.

Addition ℤ: Identity

∀a∈ℤ a + 0ℤ = a = 0ℤ + a

  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨a1 + 0, a2 + 0⟩ :: Definition +
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨a1, a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨0 + a1, 0 + a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨0, 0⟩ = ⟨0, 0⟩ + ⟨a1, a2⟩ :: Definition +

Addition ℤ: Associative

∀a,b,c∈ℤ (a + b) + c = a + (b + c)

  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨(a1 + b1) + c1, (a2 + b2) + c2)⟩ :: Definition +
  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨a1 + (b1 + c1), a2 + (b2 + c2)⟩ :: Addition ℕ
  • (⟨a1, a2⟩ + ⟨b1, b2⟩) + ⟨c1, c2⟩ =
    ⟨a1, a2⟩ + (⟨b1, b2⟩ + ⟨c1, c2)⟩ :: Definition +
As you can see, all of the heavy lifting is done by natural number addition.

Addition ℤ: Commutative

∀a,b∈ℤ a + b = b + a

  • ⟨a1, a2⟩ + ⟨b1, b2⟩ = ⟨a1 + b1, a2 + b2⟩ :: Definition
  • ⟨a1, a2⟩ + ⟨b1, b2⟩ = ⟨b1 + a1, b2 + a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨b1, b2⟩ = ⟨b1, b2⟩ + ⟨a1, a2⟩ :: Definition

∀a∈ℤ ∃(-a)∈ℤ a + (-a) = 0ℤ

∀a=⟨a1, a2⟩∈ℤ (-a) ≝ ⟨a2, a1⟩

  • ⟨a1, a2⟩ + ⟨a2, a1⟩ = ⟨a1 + a2, a2 + a1⟩ :: Definition
  • ⟨a1, a2⟩ + ⟨a2, a1⟩ = ⟨a1 + a2, a1 + a2⟩ :: Addition ℕ
  • ⟨a1, a2⟩ + ⟨a2, a1⟩ = 0ℤ

  • Definition: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
    ⟨a1, a2⟩ + ⟨b1, b2⟩ ≝ ⟨a1 + b1, a2 + b2⟩

  • Identity: ∀a∈ℤ a + 0ℤ = a = 0ℤ + a

∀a,b∈ℤ a - b ≝ a + (-b)

Subtraction ℤ Example: 3 - 2

Here's another example, this time subtracting a larger integer from a smaller one.

Subtraction ℤ: Closed

∀a,b∈ℤ a - b ∈ℤ

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℤ
⟨a1, a2⟩ - ⟨b1, b2⟩ = ⟨a1 + b2, a2 + b1⟩

Subtraction is closed because the result is always an ordered pair of natural numbers and therefore always another integer.

Subtraction ℤ: Right Identity

∀a∈ℤ a - 0ℤ = a

  • ⟨a1, a2⟩ - ⟨0, 0⟩ = ⟨a1, a2⟩ + ⟨0, 0⟩ :: Definition -
  • ⟨a1, a2⟩ - ⟨0, 0⟩ = ⟨a1 + 0, a2 + 0⟩ :: Definition +
  • ⟨a1, a2⟩ - ⟨0, 0⟩ = ⟨a1, a2⟩ :: Addition ℕ

Subtracting zero from the right leaves the number on the left unchanged.

∀a∈ℤ 0ℤ - a ≟ a

  • ⟨0, 0⟩ - ⟨a1, a2⟩ = ⟨0, 0⟩ + ⟨a2, a1⟩ :: Definition -
  • ⟨0, 0⟩ - ⟨a1, a2⟩ = ⟨0 + a2, 0 + a1⟩ :: Definition +
  • ⟨0, 0⟩ - ⟨a1, a2⟩ = ⟨a2, a1⟩ :: Arithmetic ℕ
  • ⟨a1, a2⟩ ≠ ⟨a2, a1⟩

Subtracting from zero gives the additive inverse, not the element itself, so there's no left identity.

Subtraction ℤ: Associative?

∀a,b,c∈ℤ (a - b) - c ≟ a - (b - c)

  • (a + (-b)) + (-c) ≟ a + -(b + (-c)) :: Definition -
  • (a + (-b)) + (-c) ≟ a + ((-b) + c) :: Distributive
  • (a + (-b)) + (-c) ≟ (a + (-b)) + c :: Associative +
  • -(a + (-b)) + (a + (-b)) + (-c) ≟
    -(a + (-b)) + (a + (-b)) + c :: Equality
  • -c ≟ c :: Inverses +

Subtraction is not generally associative.

Subtraction ℤ: Commutative?

∀a,b∈ℤ a - b ≟ b - a

  • a + (-b) ≟ b + (-a) :: Definition -
  • a + (-b) ≟ (-a) + b :: Associative +
  • a + (-b) ≟ -(a + (-b)) :: Distributive

Subtraction is not generally commutative. (Actually it's anticommutative: a - b = -(b - a))

Subtraction ℤ: Inverses

∀a∈ℤ a - a = 0ℤ

  • a - a = a + (-a) :: Definition -
  • a - a = 0ℤ :: Inverses +

Every integer is its own subtractive inverse.

Subtraction ℤ

  • Definition: ∀a,b∈ℤ a - b ≝ a + (-b)
  • Closed: ∀a,b∈ℤ a - b ∈ℤ
  • Right Identity: ∀a∈ℤ a + 0ℤ = a
  • Inverses: ∀a∈ℤ a - a = 0ℤ

Rational Numbers ℚ

ℚ ≝ { ⟨a, b⟩∈ℤ⊗ℤ : b > 0ℤ }

[a / b] ≝ { ⟨c1, c2⟩∈ℤ⊗ℤ : c2 > 0ℤ ∧ (c1 × b = c2 × a) }

Rational numbers extend the integers. They introduce inverses for multiplication and can represent fractional quantities.

Rational Numbers ℚ: Equality

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℚ
(⟨a1, a2⟩ =ℚ ⟨b1,b2⟩) ≝ (a1 × b2 =ℤ a2 × b1)

  • Reflexive: ∀⟨a1, a2⟩∈ℚ a1 × a2 =ℤ a2 × a1
  • Symmetric: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℚ
    a1 × b2 =ℤ a2 × b1 ⟺ b1 × a2 =ℤ b2 × a1
  • Transitive: ∀⟨a1, a2⟩,⟨b1, b2⟩,⟨c1, c2⟩∈ℚ
    (a1 × b2 =ℤ a2 × b1) ∧ (b1 × c2 =ℤ b2 × c1) ⟺
    (a1 × c2 =ℤ a2 × c1)
Rational number equality is determined by cross multiplying the members of the ordered pairs. It should be clear that this is reflexive since integer multiplication is commutative. In the same way, we can swap the equality and use commutative integer multiplication to show that this is symmetric. Transitivity is more complex but hopefully it's clear enough.

∀⟨a1, a2⟩,⟨b1, b2⟩∈ℚ

  • Antisymmetric: ∀⟨a1, a2⟩,⟨b1, b2⟩∈ℚ (a1 × b2 ≤ℤ a2 × b1) ∧ (b1 × a2 ≤ℤ b2 × a1) ⟹ (a1 × b2 =ℤ a2 × b1)

Addition ℚ

Let's define addition which maps two rational numbers to another. This will preserve all properties of integer addition, including inverses:

+: ℚ⊗ℚ ⟶ ℚ

∀a,b∈ℚ a + b ≝ ⟨a1, a2⟩ + ⟨b1, b2⟩ =
⟨(a1 × b2) + (a2 × b1), a2 × b2⟩

Addition ℚ Example: 3 + 1/2

  • 3ℚ + 1/2ℚ = ⟨3, 1⟩ + ⟨1, 2⟩
  • 3ℚ + 1/2ℚ = ⟨(3 × 2) + (1 × 1), 1 × 2⟩
  • 3ℚ + 1/2ℚ = ⟨6 + 1, 2⟩
  • 3ℚ + 1/2ℚ = ⟨7, 2⟩
  • 3ℚ + 1/2ℚ = 7/2ℚ

Addition ℚ: Isomorphic

∀ a,b∈ℤ ⟨a, 1ℤ⟩ + ⟨b, 1ℤ⟩ =
⟨(a × 1ℤ) + (1ℤ × b), 1ℤ × 1ℤ⟩ = ⟨a + b, 1ℤ⟩

Adding any two rational numbers that match integers results in the integer their sum would match.

Addition ℚ: Closed

∀a,b∈ℚ a + b ∈ℚ

∀a,b∈ℚ a + b ≝ ⟨a1, a2⟩ + ⟨b1, b2⟩ =
⟨(a1 × b2) + (a2 × b1), a2 × b2⟩

Addition for rational numbers is constructed entirely from integer arithmetic, which is closed. The product of two positive numbers is always positve so second element is positive.

We have to be a bit more careful here because not all ordered pairs of integers are rational numbers. The second member must be greater than zero.

Addition ℚ: Identity

∀a∈ℚ a + 0ℚ = a = 0ℚ + a

  • ⟨a1, a2⟩ + ⟨0, 1⟩ = ⟨(a1 × 1) + (a2 × 0), a2 × 1⟩ :: Definition +

  • ⟨a1, a2⟩ + ⟨0, 1⟩ = ⟨(0 × a2) + (1 × a1), 1 × a2⟩ :: Arithmetic ℤ
  • ⟨a1, a2⟩ + ⟨0, 1⟩ = ⟨0, 1⟩ + ⟨a1,a2⟩ :: Definition +

∀a,b,c∈ℚ (a + b) + c = a + (b + c)

Metric Space

A metric space consists of a set of points and a distance function: ⟨ℙ, d: ℙ ⊗ ℙ ⟶ ℝ⟩

The point at infinity is always part of a flat. This is what distinguishes them from rounds.

A multivector component consists of a real number and an integer tuple in an ordered pair. The tuple may be empty or have any number of integers.

This defines a multivector component. The simplest component is just an ordered pair with a real number and the empty set, but we can prepend integers to make a tuple of arbitrary length.

Components 𝕄 : Equality

  • ∀a,b∈ℝ (⟨a, {}⟩ =𝕄 ⟨b, {}⟩) ⟺ (a =ℝ b)
  • ∀i∈ℤ ∀⟨a, at⟩,⟨b, bt⟩∈𝕄

  • ∀a∈ℝ ∀i>0∈ℤ ⟨a, ⟨i, ⟨i, t⟩⟩⟩ =𝕄 ⟨a, t⟩
  • ∀a∈ℝ ∀i<0∈ℤ ⟨a, ⟨i, ⟨i, t⟩⟩⟩ =𝕄 ⟨-a, t⟩
  • ∀a∈ℝ ∀i,j∈ℤ (i ≠ j) ⟹
    ⟨a, ⟨i, ⟨j, t⟩⟩⟩ =𝕄 ⟨-a, ⟨j, ⟨i, t⟩⟩⟩

Equal integers can be combined. Unequal integers can be swapped by negating the real number.

These are the ways we can transform the components while preserving equality. When the tuples of both components are empty, component equality is the same as real number equality for the first element. The second rule shows that we can prepend the same integer to both tuples without changing equality. The third and fourth rules allow us to collapse adjacent pairs of integers in the tuple. If these integers are negative we must flip the sign of the real number. Finally, the last rule allows us to swap pairs of integers as long as we flip the sign of the real number.

Multivector 𝔾

  • ∀⟨a,t⟩∈𝕄 ⟨⟨a,t⟩, {}⟩ ∈𝔾

A multivector is a tuple consisting of one or more multivector components.

A multivector can be constructed in three ways. One is to convert a real number by making it the first member of an ordered pair and using the empty set as the second. Another is to take an existing ordered pair multivector and prepend an integer to the tuple. And finally we can create the sum of any two multivectors.

Multivectors the same if their components are the same. Swapping is allowed and components with matching tuples can be merged.

Multivector equality follows these rules. Any components with a zero value real number can be dropped. If the tuples of two adjacent components are the same they can be combined into a single component. If the first component of two multivectors have component equality and the rest of the multivector is the same, then the multivectors are equal. Two adjacent components of a multivector can be swapped without changing equality.
